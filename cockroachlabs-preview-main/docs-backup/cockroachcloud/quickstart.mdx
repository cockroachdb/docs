## Create a free trial cluster

<Steps>
  <Step>
   [Create a CockroachDB Cloud account](https://www.cockroachlabs.com/docs/cockroachcloud/create-an-account). <Info>If this is your first CockroachDB Cloud organization, it will be credited with \$400 in [free trial credits](https://www.cockroachlabs.com/docs/cockroachcloud/free-trial) to get you started.</Info>
  </Step>
  <Step>
  On the **Get Started** page, click **Create cluster**.
  </Step>
  <Step>
  On the **Select a plan** page, select **Standard**.
  </Step>
  <Step>
  On the **Cloud & Regions** page, select a cloud provider (GCP or AWS).
  </Step>
  <Step>
  In the **Regions** section, select a region for the cluster. Refer to [CockroachDB Cloud Regions](https://www.cockroachlabs.com/docs/cockroachcloud/regions) for the regions where CockroachDB Standard clusters can be deployed. To create a multi-region cluster, click **Add region** and select additional regions.
  </Step>
  <Step>
  Click **Next: Capacity**.
  </Step>
  <Step>
  On the **Capacity** page, keep the [**Provisioned capacity**](https://www.cockroachlabs.com/docs/cockroachcloud/plan-your-cluster) at the default value of 2 vCPUs.

  Click **Next: Finalize**.
  </Step>
  <Step>
  On the **Finalize** page, name your cluster. If an active free trial is listed in the right pane, you will not need to add a payment method, though you will need to do this by the [end of the trial](https://www.cockroachlabs.com/docs/cockroachcloud/free-trial#add-payment-methods) to maintain your organization's clusters.

  Click **Create cluster**.

	Your cluster will be created in a few seconds and the **Create SQL user** dialog will display.

  </Step>
</Steps>

## Create a SQL user

The **Create SQL user** dialog allows you to create a new SQL user and password.

<Steps>
  <Step>
  Enter a username in the **SQL user** field or use the one provided by default.
  </Step>
  <Step>
  Click **Generate & save password**.
  </Step>
  <Step>
  Copy the generated password and save it in a secure location.
  </Step>
  <Step>
  Click **Next**.

  	Currently, all new SQL users are created with admin privileges. For more information and to change the default settings, see [Managing SQL users on a cluster](https://www.cockroachlabs.com/docs/cockroachcloud/managing-access#manage-sql-users-on-a-cluster).
  </Step>
</Steps>

## Connect to the cluster

Select a language to connect a sample application to your cluster. To connect to your cluster directly from the command line, refer to [Connect to a Standard Cluster](https://www.cockroachlabs.com/docs/cockroachcloud/connect-to-your-cluster).


<Tabs>
  <Tab title="Java" icon="java">
Once you create a SQL user, the **Connect to cluster** dialog will show information about how to connect to your cluster.

<Steps>
  <Step>
  Select **Java** from the **Select option/language** dropdown.
  </Step>
  <Step>
  Copy the `JDBC_DATABASE_URL` environment variable command provided and save it in a secure location.

  <Note>
  **Note:**

	The connection string is pre-populated with your username, password, cluster name, and other details. Your password, in particular, will be provided *only* once. Save it in a secure place (Cockroach Labs recommends a password manager) to connect to your cluster in the future. If you forget your password, a Cluster Admin can reset it. Refer to: [Managing SQL users on a cluster](https://www.cockroachlabs.com/docs/cockroachcloud/managing-access#manage-sql-users-on-a-cluster)
  </Note>
  </Step>
</Steps>
</Tab>
<Tab title="Node.js" icon="node-js">
  Once you create a SQL user, the **Connect to cluster** dialog will show information about how to connect to your cluster.

<Steps>
  <Step>
  Select **General connection string** from the **Select option** dropdown.
  </Step>
  <Step>
  Open the **General connection string** section, then copy the connection string provided and save it in a secure location.

  <Note>
  **Note:**

	The connection string is pre-populated with your username, password, cluster name, and other details. Your password, in particular, will be provided *only* once. Save it in a secure place (Cockroach Labs recommends a password manager) to connect to your cluster in the future. If you forget your password, a Cluster Admin can reset it. Refer to: [Managing SQL users on a cluster](https://www.cockroachlabs.com/docs/cockroachcloud/managing-access#manage-sql-users-on-a-cluster)
  </Note>
  </Step>
</Steps>
</Tab>
</Tabs>


## Configure the connection environment variable

<Tabs>
  <Tab title="Mac" icon="apple">

In a terminal set the `DATABASE_URL` environment variable to the connection string:

```bash
export DATABASE_URL="<connection-string>"
```

The code sample uses the connection string stored in the environment variable `DATABASE_URL` to connect to your cluster.
</Tab>
<Tab title="Linux" icon="linux">
In a terminal set the `DATABASE_URL` environment variable to the connection string:

```bash
export DATABASE_URL="<connection-string>"
```

The code sample uses the connection string stored in the environment variable `DATABASE_URL` to connect to your cluster.
</Tab>
<Tab title="Windows" icon="windows">
In a terminal set the `DATABASE_URL` environment variable to the connection string:

```bash
$env:DATABASE_URL = "<connection-string>"
```

The code sample uses the connection string stored in the environment variable `DATABASE_URL` to connect to your cluster.
</Tab>
</Tabs>

## Run the Node.js sample code

<Steps>
  <Step>
   Clone the `quickstart-code-samples` repo:

	```bash
	git clone https://github.com/cockroachdb/quickstart-code-samples
	```
  </Step>
  <Step>
   Navigate to the `node` directory of the repo:

	```bash
	cd quickstart-code-samples/node
	```

	The code sample in this directory does the following:
	1. Connects to CockroachDB Cloud with the [node-postgres driver](https://node-postgres.com/) using the connection string set in the `DATABASE_URL` environment variable.
	2. Creates a table.
	3. Inserts some data into the table.
	4. Reads the inserted data.
	5. Prints the data to the terminal.
  </Step>
  <Step>
   Install the app requirements:

	```bash
	npm install
	```
  </Step>
  <Step>
   Run the app:

	```bash
	node app.js
	```

  	The output will look like this:
	```bash
	Hello world!
	```
  </Step>
</Steps>

## Learn more

Now that you have a CockroachDB Standard cluster running, try out the following:

- Build a simple CRUD application in [Go](https://www.cockroachlabs.com/docs/v25.3/build-a-go-app-with-cockroachdb), [Java](https://www.cockroachlabs.com/docs/v25.3/build-a-java-app-with-cockroachdb), [Node.js](https://www.cockroachlabs.com/docs/v25.3/build-a-nodejs-app-with-cockroachdb), or [Python](https://www.cockroachlabs.com/docs/v25.3/build-a-python-app-with-cockroachdb).
- [Learn CockroachDB SQL](https://www.cockroachlabs.com/docs/cockroachcloud/learn-cockroachdb-sql).
- [Create and manage SQL users](https://www.cockroachlabs.com/docs/cockroachcloud/managing-access).
- Explore our [example apps](https://www.cockroachlabs.com/docs/v25.3/example-apps) for examples on how to build applications using your preferred driver or ORM and run it on CockroachDB.
- [Migrate your existing data](https://www.cockroachlabs.com/docs/molt/migration-overview).

This page highlights just one way you can get started with CockroachDB. For information on other options that are available when creating a CockroachDB cluster, see the following:

- To create a Self-Hosted cluster, see [Start a Local Cluster](https://www.cockroachlabs.com/docs/v25.3/start-a-local-cluster).
- To create a CockroachDB Advanced cluster, see [Quickstart with CockroachDB Advanced](https://www.cockroachlabs.com/docs/cockroachcloud/quickstart-trial-cluster).
- To create a CockroachDB Standard cluster with other configurations (e.g., a different cloud provider, region, or provisioned capacity), see [Create a CockroachDB Standard Cluster](https://www.cockroachlabs.com/docs/cockroachcloud/create-your-cluster).
- To connect to a CockroachDB Standard cluster with other options (e.g., a different SQL user) and connection methods (with an application or [CockroachDB compatible tool](https://www.cockroachlabs.com/docs/v25.3/third-party-database-tools)), see [Connect to a CockroachDB Standard Cluster](https://www.cockroachlabs.com/docs/cockroachcloud/connect-to-your-cluster).
