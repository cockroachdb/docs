This quickstart guides you through creating, validating, and deploying your first <a href="/docs/VERSION_PLACEHOLDER/molt/userscript-overview" data-versioned="true">userscript</a> based on a <a href="/docs/VERSION_PLACEHOLDER/molt/userscript-cookbook" data-versioned="true">cookbook</a> example.

## Before you begin

- <a href="/docs/VERSION_PLACEHOLDER/molt/molt-replicator#installation" data-versioned="true">Install MOLT Replicator **v1.3.0 or later**</a> for full compatibility with the userscript API.
- Install a TypeScript-compatible IDE (for example, [VS Code](https://code.visualstudio.com/) ).

## Step 1: Set up your environment

Create a new TypeScript file for your userscript in your IDE (for example, `userscript.ts`).

For IDE autocomplete and type-checking support, optionally add the Replicator TypeScript definitions. For details, refer to <a href="/docs/VERSION_PLACEHOLDER/molt/userscript-api#access-this-api" data-versioned="true">Access this API</a>.

## Step 2: Write a userscript

Use the <a href="/docs/VERSION_PLACEHOLDER/molt/userscript-cookbook" data-versioned="true">userscript cookbook</a> as a starting point. The cookbook provides ready-to-use examples that you can copy and adapt.

Copy and adapt a cookbook example that matches your use case. For example, adapt the template to <a href="/docs/VERSION_PLACEHOLDER/molt/userscript-cookbook#filter-a-single-table" data-versioned="true">filter a single table</a> to exclude the `audit_logs` table from the `public` schema during replication:

```ts
import * as api from "replicator@v2";

const TARGET_SCHEMA_NAME = "defaultdb.public";
const TABLE_TO_SKIP = "audit_logs";

api.configureTargetSchema(SCHEMA_NAME, {
  onRowUpsert: (row, meta) => {
    if (meta.table === TABLE_TO_SKIP) {
      return null; // Skip this table
    }
    return row;
  },
  onRowDelete: (row, meta) => {
    if (meta.table === TABLE_TO_SKIP) {
      return null;
    }
    return row;
  }
});
```

For details on the userscript API and handler functions, refer to the <a href="/docs/VERSION_PLACEHOLDER/molt/userscript-api" data-versioned="true">Userscript API</a>.

<Tip>
When writing userscripts, follow the <a href="/docs/VERSION_PLACEHOLDER/molt/userscript-api#best-practices" data-versioned="true">best practices</a>.
</Tip>

## Step 3: Use with Replicator

Include the <a href="/docs/VERSION_PLACEHOLDER/molt/replicator-flags#userscript" data-versioned="true">`--userscript`</a> flag to have <a href="/docs/VERSION_PLACEHOLDER/molt/molt-replicator" data-versioned="true">MOLT Replicator</a> apply the userscript. For example, to apply your userscript during PostgreSQL replication:

```shell
replicator pglogical \
--sourceConn $SOURCE \
--targetConn $TARGET \
--targetSchema defaultdb.public \
--slotName molt_slot \
--publicationName molt_fetch \
--stagingSchema defaultdb._replicator \
--stagingCreateSchema \
--userscript userscript.ts
```
