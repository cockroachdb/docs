The <a href="/docs/VERSION_PLACEHOLDER/kubernetes-overview" data-versioned="true">Public Kubernetes operator</a> allows you to configure, deploy, and manage CockroachDB self-hosted clusters on Kubernetes. The Public operator is released on a separate schedule and is versioned independently from CockroachDB. To learn more about different approaches to deploy CockroachDB using Kubernetes, refer to <a href="/docs/VERSION_PLACEHOLDER/kubernetes-overview" data-versioned="true">Kubernetes Overview</a>.

<Warning>
New deployments of CockroachDB on Kubernetes are recommended to use the newer, fully-featured CockroachDB operator that's easier to deploy and supports scaling of multi-region clusters. To migrate an existing deployment to use the CockroachDB operator, read the <a href="/docs/VERSION_PLACEHOLDER/migrate-cockroachdb-kubernetes-helm" data-versioned="true">Helm</a> and <a href="/docs/VERSION_PLACEHOLDER/migrate-cockroachdb-kubernetes-operator" data-versioned="true">Public operator</a> migration guides.
</Warning>

This page announces releases of the Public operator and provides links to more information on GitHub. **Version 2.18.3 is the latest release**.

In addition to monitoring this page, you can subscribe to be notified about releases to the public. Visit the [Public operator source code repository](https://github.com/cockroachdb/cockroach-operator) and click **Watch**.

<Tip>
If you already use the [Helm](https://helm.sh/) package manager to manage your Kubernetes infrastructure, you can manage CockroachDB self-hosted clusters by using the [public Helm chart](https://github.com/cockroachdb/helm-charts/tree/master/cockroachdb) instead of using the Public operator. The Helm chart does not use the Public operator, and there is no automated way to migrate from one method to the other.

To be notified about updates to the Helm chart, visit the [public Helm chart source code repository](https://github.com/cockroachdb/helm-charts/tree/master/cockroachdb) and click **Watch**.
</Tip>
