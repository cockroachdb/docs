</Note>

This page has instructions for migrating data from a CockroachDB Standard cluster to a CockroachDB Advanced cluster, by exporting to CSV and using [`IMPORT INTO`](/docs/v25.4/import-into). You may want to migrate to CockroachDB Advanced if you want a single-tenant cluster with no shared resources.

The following steps use sample data from the [`tpcc` workload](/docs/v25.4/cockroach-workload#workloads).

## Before you begin

These instructions assume you already have the following:

- A [CockroachDB Standard cluster](/docs/cockroachcloud/quickstart) from which you want to migrate data.
- A [CockroachDB Advanced cluster](/docs/cockroachcloud/quickstart-trial-cluster).
- [Cloud storage](/docs/v25.4/use-cloud-storage).

## Step 1. Export data to cloud storage

First, upload your CockroachDB Standard data to a cloud storage location where the CockroachDB Advanced cluster can access it.

1. [Connect to your CockroachDB Standard cluster](/docs/cockroachcloud/connect-to-your-cluster) and run the [`EXPORT`](/docs/v25.4/export) statement for each table you need to migrate. For example, the following statement exports the `warehouse` table from the [`tpcc`](/docs/v25.4/cockroach-workload#workloads) database to an Amazon S3 bucket:

    ```sql
    EXPORT INTO CSV
      's3://{BUCKET NAME}/migration-data?AWS_ACCESS_KEY_ID={ACCESS KEY}&AWS_SECRET_ACCESS_KEY={SECRET ACCESS KEY}'
      FROM TABLE tpcc.warehouse;
    ```

    This command uploads the table in CSV format to a `migration-data` directory in the S3 bucket.

    The output shows the programmatically generated filename, which you'll reference when you [import the CSV](#step-2-import-the-csv):

    ```text
                                  filename                             | rows | bytes
    -------------------------------------------------------------------+------+--------
      export1732f39b0dbe4e2b0000000000000001-n824429310719852545.0.csv |    1 |    43
    ```

    The export file contains the following CSV data:

    ```text
    0,8,17,13,11,SF,640911111,0.0806,300000.00
    ```
2. Repeat this step for each table you want to migrate. For example, export one more table (`district`) from the `tpcc` database:

    ```sql
    EXPORT INTO CSV
      's3://{BUCKET NAME}/migration-data?AWS_ACCESS_KEY_ID={ACCESS KEY}&AWS_SECRET_ACCESS_KEY={SECRET ACCESS KEY}'
      FROM TABLE tpcc.district;
    ```

## Step 2. Import the CSV

<Tip>
For best practices for optimizing import performance in CockroachDB, refer to [Import Performance Best Practices](/docs/v25.4/import-performance-best-practices).
</Tip>

1. [Connect to your CockroachDB Advanced cluster](/docs/cockroachcloud/connect-to-your-cluster) and [create the database](/docs/v25.4/create-database) you want to import the tables into. For example:

    ```sql
    CREATE DATABASE tpcc;
    ```
2. Write a [`CREATE TABLE`](/docs/v25.4/create-table) statement that matches the schema of the table data you're importing.

    <Tip>
    You can use the [`SHOW CREATE TABLE`](/docs/v25.4/show-create#show-the-create-table-statement-for-a-table) statement in the CockroachDB Standard cluster to view the `CREATE` statement for the table you're migrating.
    </Tip>

    <Note>
    The column order in your schema must match the column order in the file being imported.
    </Note>

    For example, to import the `tpcc.warehouse` data into a `warehouse` table, issue the following statement to create the new table:

    ```sql
    CREATE TABLE tpcc.warehouse (
      w_id INT8 NOT NULL,
      w_name VARCHAR(10) NULL,
      w_street_1 VARCHAR(20) NULL,
      w_street_2 VARCHAR(20) NULL,
      w_city VARCHAR(20) NULL,
      w_state CHAR(2) NULL,
      w_zip CHAR(9) NULL,
      w_tax DECIMAL(4,4) NULL,
      w_ytd DECIMAL(12,2) NULL,
      CONSTRAINT "primary" PRIMARY KEY (w_id ASC),
      FAMILY "primary" (w_id, w_name, w_street_1, w_street_2, w_city, w_state, w_zip, w_tax, w_ytd)
    );
    ```

    Next, use `IMPORT INTO` to import the data into the new table, specifying the filename of the export from [Step 1](#step-1-export-data-to-cloud-storage):

    ```sql
    IMPORT INTO tpcc.warehouse (w_id, w_name, w_street_1, w_street_2, w_city, w_state, w_zip, w_tax, w_ytd)
      CSV DATA ('s3://{BUCKET NAME}/migration-data/{EXPORT FILENAME}?AWS_ACCESS_KEY_ID={ACCESS_KEY}&AWS_SECRET_ACCESS_KEY={SECRET_KEY}')
    ```

    ```text
            job_id       |  status   | fraction_completed | rows | index_entries | bytes
    ---------------------+-----------+--------------------+------+---------------+--------
      652285202857820161 | succeeded |                  1 |   10 |             0 |  1017
    (1 row)
    ```
3. Repeat the previous command for each CSV file you want to import. For example, import the `tpcc.district` data:

    Issue the following statement to create a new `district` table:

    ```sql
    CREATE TABLE tpcc.district (
      d_id INT8 NOT NULL,
      d_w_id INT8 NOT NULL,
      d_name VARCHAR(10) NULL,
      d_street_1 VARCHAR(20) NULL,
      d_street_2 VARCHAR(20) NULL,
      d_city VARCHAR(20) NULL,
      d_state CHAR(2) NULL,
      d_zip CHAR(9) NULL,
      d_tax DECIMAL(4,4) NULL,
      d_ytd DECIMAL(12,2) NULL,
      d_next_o_id INT8 NULL,
      CONSTRAINT "primary" PRIMARY KEY (d_w_id ASC, d_id ASC),
      FAMILY "primary" (d_id, d_w_id, d_name, d_street_1, d_street_2, d_city, d_state, d_zip, d_tax, d_ytd, d_next_o_id)
    );
    ```

    Next, use `IMPORT INTO` to import the data into the new table, specifying the filename of the export from [Step 1](#step-1-export-data-to-cloud-storage):

    ```sql
    IMPORT INTO tpcc.district (d_id, d_w_id, d_name, d_street_1, d_street_2, d_city, d_state, d_zip, d_tax, d_ytd, d_next_o_id)
      CSV DATA ('s3://{BUCKET NAME}/migration-data/{EXPORT FILENAME}?AWS_ACCESS_KEY_ID={ACCESS_KEY}&AWS_SECRET_ACCESS_KEY={SECRET_KEY}');
    ```

    ```text
            job_id       |  status   | fraction_completed | rows | index_entries | bytes
    ---------------------+-----------+--------------------+------+---------------+--------
      824426026443571201 | succeeded |                  1 |   10 |             0 |  1067
    (1 row)
    ```
4. **(Optional)** To verify that the data was imported, use [`SHOW TABLES`](/docs/v25.4/show-tables):

    ```sql
    SHOW TABLES FROM tpcc;
    ```

    ```text
      schema_name | table_name | type  | owner | estimated_row_count | locality
    --------------+------------+-------+-------+---------------------+-----------
      public      | district   | table | ryan  |                  10 | NULL
      public      | warehouse  | table | ryan  |                   1 | NULL
    (2 rows)
    ```

## Step 3. Add any foreign key relationships

Once all of the tables you want to migrate have been imported into the CockroachDB Advanced cluster, add the [foreign key](/docs/v25.4/foreign-key) relationships. To do this, use [`ALTER TABLE... ADD CONSTRAINT`](/docs/v25.4/alter-table#add-constraint). For example:

```sql
ALTER TABLE tpcc.district ADD CONSTRAINT fk_d_w_id_ref_warehouse FOREIGN KEY (d_w_id) REFERENCES tpcc.warehouse(w_id);
```
