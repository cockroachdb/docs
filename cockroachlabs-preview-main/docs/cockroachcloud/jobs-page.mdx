The **Jobs** page of the CockroachDB Cloud Console provides details of all jobs in the cluster.

Viewing jobs requires the [Cluster Monitor](/docs/cockroachcloud/authorization#cluster-monitor), [Cluster Operator](/docs/cockroachcloud/authorization#cluster-operator), or [Cluster Admin](/docs/cockroachcloud/authorization#cluster-admin) role. To view this page, select a cluster from the [**Clusters** page](/docs/cockroachcloud/cluster-management#view-clusters-page), and click **Jobs** in the **Monitoring** section of the left side navigation.

<Note>
By default, jobs are deleted every 14 days. You can change this interval using the `jobs.retention_time` [cluster setting](/docs/v25.4/cluster-settings).

The Jobs list is designed for you to manage pending work. It is not intended to display the canonical record of all jobs that have run. If you need a historical record of all jobs you have run, you should log this information externally.
</Note>

## Jobs table

Use the **Jobs** table to see recently created and completed jobs.

  ![DB Console Jobs Page](/images/v25.4/ui-jobs-page.png)
</Frame>

### Filter jobs

Use the controls to filter jobs:

- **Status**: filter jobs by [job status](#job-status).
- **Type**: filter jobs by type. [Automatic table statistics](/docs/v25.4/cost-based-optimizer#table-statistics) jobs are not displayed even when the **Type** menu is set to **All**. To view these jobs, set **Type** to **Auto-Statistics Creation**.
- **Show**: toggle between showing the latest 50 jobs or all jobs.

The status line after the filters shows the number of jobs displayed and a message indicating the oldest time (in UTC) that jobs are shown for. The oldest time is calculated as `now -` the duration of the `jobs.retention_time` [cluster setting](/docs/v25.4/cluster-settings), set to 28 days by default.

The table is paginated with 20 jobs per page. Click the page controls following the table to traverse pages.

The following columns are available for each job. Click **Columns** to select the columns to display in the table.

## Job details

The details show:

- **Job ID**
- **Status**
- **Creation Time**
- **Last Modified Time**
- **Completed Time**
- **Last Execution Time**
- **Execution Count**
- **User Name**
- error messages (if any)
