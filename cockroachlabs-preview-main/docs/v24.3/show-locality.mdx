The `SHOW LOCALITY` [statement](/docs/v24.3/sql-statements) returns the [locality](/docs/v24.3/cockroach-start#locality) of the current node.

If locality was not specified on node startup, the statement returns an empty row.

## Required privileges

No [privileges](/docs/v24.3/security-reference/authorization#managing-privileges) are required to list the locality of the current node.

## Example

### Setup

The following example uses MovR, a fictional vehicle-sharing application, to demonstrate CockroachDB SQL statements. For more information about the MovR example application and dataset, see [MovR: A Global Vehicle-sharing App](/docs/v24.3/movr).

To follow along, run [`cockroach demo movr`](/docs/v24.3/cockroach-demo) with the `--nodes` and `--demo-locality` tags. This command opens an interactive SQL shell to a temporary, multi-node in-memory cluster with the `movr` database preloaded and set as the [current database](/docs/v24.3/sql-name-resolution#current-database).

```shell
$ cockroach demo movr --nodes=3 --demo-locality=region=us-east,az=a:region=us-central,az=b:region=us-west1,az=c
```

### Show locality

```sql
> SHOW LOCALITY;
```

```text
       locality
+---------------------+
  region=us-east,az=a
(1 row)
```

### Show locality with a built-in function

If you know the locality key, you can use the [`crdb_internal.locality_value`](/docs/v24.3/functions-and-operators#system-info-functions) built-in function to return the locality value for the current node:

```sql
> SELECT * FROM crdb_internal.locality_value('region');
```

```text
  crdb_internal.locality_value
+------------------------------+
  us-east
(1 row)
```

```sql
> SELECT * FROM crdb_internal.locality_value('az');
```

```text
  crdb_internal.locality_value
+------------------------------+
  a
(1 row)
```

For a more extensive example, see [Create a table with node locality information](/docs/v24.3/cockroach-start#create-a-table-with-node-locality-information).
