The `SHOW SCHEMAS` [statement](/docs/v24.3/sql-statements) lists all [schemas](/docs/v24.3/sql-name-resolution#naming-hierarchy) in a database.

## Required privileges

The `CONNECT` [privilege](/docs/v24.3/security-reference/authorization#managing-privileges) on the database is required to list the schemas in a database.

## Example

#### Setup

To follow along, run [`cockroach demo`](/docs/v24.3/cockroach-demo) to start a temporary, in-memory cluster with the [`movr`](/docs/v24.3/movr) sample dataset preloaded:

```shell
$ cockroach demo
```

### Show schemas in the current database

```sql
> CREATE SCHEMA org_one;
```

```sql
> SHOW SCHEMAS;
```

```text
     schema_name
----------------------
  crdb_internal
  information_schema
  org_one
  pg_catalog
  pg_extension
  public
(6 rows)
```
