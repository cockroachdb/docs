The `SHOW SCHEDULES` [statement](/docs/v24.3/sql-statements) lists all of the currently active [backup schedules](/docs/v24.3/create-schedule-for-backup) and [changefeed schedules](/docs/v24.3/create-schedule-for-changefeed).

It also lists all of the currently active scheduled jobs for [Row-Level TTL](/docs/v24.3/row-level-ttl).

## Required privileges

The following users can show a schedule:

- Members of the [`admin` role](/docs/v24.3/security-reference/authorization#default-roles). By default, the `root` user belongs to the `admin` role.
- Owners of a backup schedule, i.e., the user that [created the backup schedule](/docs/v24.3/create-schedule-for-backup).
- Owners of a changefeed schedule, i.e., the user that [created the changefeed schedule](/docs/v24.3/create-schedule-for-changefeed).

## Response

The output of `SHOW SCHEDULES` is sorted by creation time. The following fields are returned for each schedule:

<table><thead><tr><th>Field</th><th>Description</th></tr></thead><tbody><tr><td><code>id</code></td><td>A unique ID to identify each schedule. This value is used if you want to control schedules (i.e., <a href="/docs/v24.3/pause-schedules">pause</a>, <a href="/docs/v24.3/resume-schedules">resume</a>, or <a href="/docs/v24.3/drop-schedules">drop</a> it).</td></tr><tr><td><code>label</code></td><td>The name used to identify the schedule, given at the time of schedule creation.</td></tr><tr><td><code>schedule&#95;status</code></td><td>The schedule's current status.</td></tr><tr><td><code>next&#95;run</code></td><td>The <a href="/docs/v24.3/timestamp"><code>TIMESTAMP</code></a> at which the next scheduled job is slated to run.</td></tr><tr><td><code>state</code></td><td>Displays last-known errors or messages about the schedule. Cleared on retry of a scheduled job.</td></tr><tr><td><code>recurrence</code></td><td>How often the scheduled job is run, which is set at the time of schedule creation.</td></tr><tr><td><code>jobsrunning</code></td><td>The number of <a href="/docs/v24.3/show-jobs">jobs</a> currently running for the schedule.</td></tr><tr><td><code>owner</code></td><td>The <a href="/docs/v24.3/create-user">user</a> who created the schedule. Users with active schedules cannot be dropped.</td></tr><tr><td><code>created</code></td><td>The <a href="/docs/v24.3/timestamp"><code>TIMESTAMP</code></a> when the job was created.</td></tr><tr><td><code>on&#95;previous&#95;running</code></td><td>The status of the scheduled job if the previous backup is still running. This <a href="/docs/v24.3/create-schedule-for-backup#schedule-options">option</a> by default is set to <code>WAIT</code>.</td></tr><tr><td><code>on&#95;execution&#95;failure</code></td><td>The status of the scheduled job if an error occurs during the job execution. This <a href="/docs/v24.3/create-schedule-for-backup#schedule-options">option</a> by default is set to <code>RETRY&#95;SCHED</code>.</td></tr><tr><td><code>command</code></td><td>The command that the scheduled job will run. This can be derived from the <a href="/docs/v24.3/create-schedule-for-backup"><code>CREATE SCHEDULE FOR BACKUP</code></a> statement, <a href="/docs/v24.3/create-schedule-for-changefeed"><code>CREATE SCHEDULE FOR CHANGEFEED</code></a> statement, or generated by the system for <a href="/docs/v24.3/row-level-ttl">Row-Level TTL</a> jobs.</td></tr><tr><td><code>backup&#95;type</code></td><td>Displays whether a scheduled backup job is <a href="/docs/v24.3/take-full-and-incremental-backups#full-backups"><code>FULL</code></a> or <a href="/docs/v24.3/take-full-and-incremental-backups#incremental-backups"><code>INCREMENTAL</code></a> when running <code>SHOW SCHEDULES FOR BACKUP</code>.</td></tr></tbody></table>
