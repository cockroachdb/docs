The [`VIEWACTIVITY`](/docs/v24.3/security-reference/authorization#viewactivity) or [`VIEWACTIVITYREDACTED`](/docs/v24.3/security-reference/authorization#viewactivityredacted) [system privileges](/docs/v24.3/security-reference/authorization#supported-privileges) **do not** grant access to this page.
</Note>

The **Hot Ranges** page of the DB Console provides details about ranges receiving a high number of reads or writes. These are known as *hot ranges*.

When [optimizing](/docs/v24.3/performance-best-practices-overview#hot-spots) or [troubleshooting](/docs/v24.3/query-behavior-troubleshooting#single-hot-node) statement performance, this page can help you identify nodes, ranges, or tables that are experiencing hot spots.

To view this page, [access the DB Console](/docs/v24.3/ui-overview#db-console-access) and click **Hot Ranges** in the left-hand navigation.

## Filter hot ranges

Use the **Filter** menu to filter the [hot ranges list](#hot-ranges-list) on any combination of: node ID, store ID, database, table, index, or locality.

## Hot ranges list

The **Hot ranges** list displays the ranges with the highest queries per second (QPS) from each node [`store`](/docs/v24.3/architecture/storage-layer).

<Note>
Hot ranges are not necessarily problematic. Some ranges naturally experience higher QPS than others. For example, a range for a frequently accessed table will have a higher QPS.

However, a significant increase in traffic can also indicate a *hot spot* on the range that should be reduced. For more information, see [Hot spots](/docs/v24.3/performance-best-practices-overview#hot-spots).
</Note>

To view the [Range Report](#range-report) for a hot range, click its range ID.

## Range Report

The **Range Report** is typically used for [advanced debugging](/docs/v24.3/ui-debug-pages#even-more-advanced-debugging) purposes.

If your aim is to [reduce hot spots](/docs/v24.3/performance-best-practices-overview#hot-spots), refer to the following fields:

- `Key Range` shows the interval of the [key space](/docs/v24.3/architecture/distribution-layer#range-descriptors) that is "hottest" (i.e., read by the processor). This is expressed as a span of key values.
- `Lease Holder QPS` shows the queries executed per second on the node that holds the [range lease](/docs/v24.3/architecture/replication-layer#leases). If a hot range is not properly using [load-based splitting](/docs/v24.3/load-based-splitting), this will be greater than the value configured by the `kv.range_split.load_qps_threshold` [cluster setting](/docs/v24.3/cluster-settings) ( `2500` by default).
