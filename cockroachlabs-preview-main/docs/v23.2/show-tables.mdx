The `SHOW TABLES` [statement](/docs/v23.2/sql-statements) lists the schema, table name, table type, owner, and estimated row count for the tables or [views](/docs/v23.2/views) in a schema or database.

<Note>
While a table or view is being [dropped](/docs/v23.2/drop-table), `SHOW TABLES` will list the object with a `(dropped)` suffix.
</Note>

## Required privileges

The `CONNECT` [privilege](/docs/v23.2/security-reference/authorization#managing-privileges) on the database of the concerned table is required to list it with SHOW TABLES.

## Parameters

<table><thead><tr><th>Parameter</th><th>Description</th></tr></thead><tbody><tr><td><code>database&#95;name</code></td><td>The name of the database for which to show tables.</td></tr><tr><td><code>schema&#95;name</code></td><td>The name of the schema for which to show tables.</td></tr></tbody></table>

When a `database_name` and `schema_name` are omitted, the tables of the [current schema](/docs/v23.2/sql-name-resolution#current-schema) in the [current database](/docs/v23.2/sql-name-resolution#current-database) are listed.

`SHOW TABLES` will attempt to find a schema with the specified name first. If that fails, it will try to find a database with that name instead, and list the tables of its `public` schema. For more details, see [Name Resolution](/docs/v23.2/sql-name-resolution).

## Performance

To optimize the performance of the `SHOW TABLES` statement, you can do the following:

- Disable table row-count estimation by setting the `sql.show_tables.estimated_row_count.enabled` [cluster setting](/docs/v23.2/cluster-settings) to `false` before executing a `SHOW TABLES` statement.
- Avoid running `SHOW TABLES` on databases with a large number of tables (e.g., more than 10,000 tables).

## Examples

### Setup

The following examples use MovR, a fictional vehicle-sharing application, to demonstrate CockroachDB SQL statements. For more information about the MovR example application and dataset, see [MovR: A Global Vehicle-sharing App](/docs/v23.2/movr).

To follow along, run [`cockroach demo`](/docs/v23.2/cockroach-demo) with the [`--nodes`](/docs/v23.2/cockroach-demo#flags) and [`--demo-locality`](/docs/v23.2/cockroach-demo#flags) flags. This command opens an interactive SQL shell to a temporary, multi-node in-memory cluster with the `movr` database preloaded and set as the [current database](/docs/v23.2/sql-name-resolution#current-database).

```shell
$ cockroach demo --nodes=6 --demo-locality=region=us-east,zone=us-east-a:region=us-east,zone=us-east-b:region=us-central,zone=us-central-a:region=us-central,zone=us-central-b:region=us-west,zone=us-west-a:region=us-west,zone=us-west-b
```

### Show tables in the current database

`SHOW TABLES` uses the [current schema](/docs/v23.2/sql-name-resolution#current-schema) `public` set by default in `search_path`:

```sql
> SHOW TABLES;
```

```text

--------------+----------------------------+-------+-------+---------------------+-----------
  public      | promo_codes                | table | demo  |                   0 | NULL
  public      | rides                      | table | demo  |                   0 | NULL
  public      | user_promo_codes           | table | demo  |                   0 | NULL
  public      | users                      | table | demo  |                   0 | NULL
  public      | vehicle_location_histories | table | demo  |                   0 | NULL
  public      | vehicles                   | table | demo  |                   0 | NULL
(6 rows)
```

### Show tables in a different schema

You can show the tables in schemas other than the current schema. You can also show the schema by table:

```sql
> SHOW TABLES FROM movr.information_schema;
```

```sql
> SHOW TABLES FROM information_schema;
```

Because `movr` is the current database, these statements return the same output:

```text
     schema_name     |              table_name               | type  | owner | estimated_row_count | locality
---------------------+---------------------------------------+-------+-------+---------------------+-----------
  information_schema | administrable_role_authorizations     | table | NULL  |                NULL | NULL
  information_schema | applicable_roles                      | table | NULL  |                NULL | NULL
  information_schema | attributes                            | table | NULL  |                NULL | NULL
  information_schema | character_sets                        | table | NULL  |                NULL | NULL
  ...
(86 rows)
```

### Show tables in a different database

You can also show tables from a different database.

```sql
> SHOW TABLES FROM system.public;
```

### Show user-defined tables with comments

You can use [`COMMENT ON`](/docs/v23.2/comment-on) to add comments on a table.

```sql
> COMMENT ON TABLE users IS 'This table contains information about users.';
```

To view a table's comments:

```sql
> SHOW TABLES FROM movr WITH COMMENT;
```

```text
  schema_name |         table_name         | type  | owner | estimated_row_count | locality |                   comment
--------------+----------------------------+-------+-------+---------------------+----------+-----------------------------------------------
  public      | promo_codes                | table | demo  |                1000 | NULL     |
  public      | rides                      | table | demo  |                 500 | NULL     |
  public      | user_promo_codes           | table | demo  |                   0 | NULL     |
  public      | users                      | table | demo  |                  50 | NULL     | This table contains information about users.
  public      | vehicle_location_histories | table | demo  |                1000 | NULL     |
  public      | vehicles                   | table | demo  |                  15 | NULL     |
(6 rows)
```

You can also view comments on a table with [`SHOW CREATE`](/docs/v23.2/show-create):

```sql
> SHOW CREATE TABLE users;
```

```text
  table_name |                             create_statement
-------------+---------------------------------------------------------------------------
  users      | CREATE TABLE users (
             |     id UUID NOT NULL,
             |     city VARCHAR NOT NULL,
             |     name VARCHAR NULL,
             |     address VARCHAR NULL,
             |     credit_card VARCHAR NULL,
             |     CONSTRAINT users_pkey PRIMARY KEY (city ASC, id ASC)
             | );
             | COMMENT ON TABLE users IS 'This table contains information about users.'
(1 row)
```

For more information, see [`COMMENT ON`](/docs/v23.2/comment-on).

### Show locality of tables

For [multi-region](/docs/v23.2/multiregion-overview) tables, you can display the locality of each table using the `SHOW TABLES` command.

1. [Set the primary region](/docs/v23.2/alter-database#set-primary-region) on `movr` to `us-east`:

    ```sql
    > ALTER DATABASE movr SET PRIMARY REGION "us-east";
    ```

    All tables will be [`REGIONAL BY TABLE`](/docs/v23.2/alter-table#set-the-table-locality-to-regional-by-table) in the primary region by default.
2. Configure the `users` table to be [`REGIONAL BY ROW`](/docs/v23.2/alter-table#set-the-table-locality-to-regional-by-row):

    ```sql
    > ALTER TABLE users SET LOCALITY REGIONAL BY ROW;
    ```

    ```sql
    > SHOW TABLES;
    ```

    ```text
      schema_name |         table_name         | type  | owner | estimated_row_count |              locality
    --------------+----------------------------+-------+-------+---------------------+--------------------------------------
      public      | promo_codes                | table | demo  |                1000 | REGIONAL BY TABLE IN PRIMARY REGION
      public      | rides                      | table | demo  |                 500 | REGIONAL BY TABLE IN PRIMARY REGION
      public      | user_promo_codes           | table | demo  |                   0 | REGIONAL BY TABLE IN PRIMARY REGION
      public      | users                      | table | demo  |                  50 | REGIONAL BY ROW
      public      | vehicle_location_histories | table | demo  |                1000 | REGIONAL BY TABLE IN PRIMARY REGION
      public      | vehicles                   | table | demo  |                  15 | REGIONAL BY TABLE IN PRIMARY REGION
    (6 rows)
    ```

    <Note>
    Locality information for tables is also available in the `locality` column within the [`crdb_internal.tables`](/docs/v23.2/crdb-internal) table.
    </Note>
