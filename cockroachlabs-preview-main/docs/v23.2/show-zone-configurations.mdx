Use the `SHOW ZONE CONFIGURATIONS` [statement](/docs/v23.2/sql-statements) to view details about existing [replication zones](/docs/v23.2/configure-replication-zones).

## Synopsis

<ShowZoneConfigurations2 />

## Required privileges

No [privileges](/docs/v23.2/security-reference/authorization#managing-privileges) are required to list replication zones.

## Parameters

<table><thead><tr><th>Parameter</th><th>Description</th></tr></thead><tbody><tr><td><code>zone&#95;name</code></td><td>The name of the system <a href="/docs/v23.2/architecture/overview#architecture-range">range</a> for which to show <a href="/docs/v23.2/configure-replication-zones">replication zone configurations</a>.</td></tr><tr><td><code>database&#95;name</code></td><td>The name of the <a href="/docs/v23.2/create-database">database</a> for which to show <a href="/docs/v23.2/configure-replication-zones">replication zone configurations</a>.</td></tr><tr><td><code>table&#95;name</code></td><td>The name of the <a href="/docs/v23.2/create-table">table</a> for which to show <a href="/docs/v23.2/configure-replication-zones">replication zone configurations</a>.</td></tr><tr><td><code>partition&#95;name</code></td><td>The name of the <a href="/docs/v23.2/partitioning">partition</a> for which to show <a href="/docs/v23.2/configure-replication-zones">replication zone configurations</a>.</td></tr><tr><td><code>index&#95;name</code></td><td>The name of the <a href="/docs/v23.2/indexes">index</a> for which to show <a href="/docs/v23.2/configure-replication-zones">replication zone configurations</a>.</td></tr></tbody></table>

## Examples

#### Setup

The following examples use MovR, a fictional vehicle-sharing application, to demonstrate CockroachDB SQL statements. For more information about the MovR example application and dataset, see [MovR: A Global Vehicle-sharing App](/docs/v23.2/movr).

To follow along, run [`cockroach demo`](/docs/v23.2/cockroach-demo) with the `--geo-partitioned-replicas` flag. This command opens an interactive SQL shell to a temporary, 9-node in-memory cluster with the `movr` database.

```shell
$ cockroach demo --geo-partitioned-replicas
```

### View the replication zone for a database

To control replication for a specific database, use the `ALTER DATABASE... CONFIGURE ZONE` statement to define the relevant values (other values will be inherited from the parent zone):

```sql
> ALTER DATABASE movr CONFIGURE ZONE USING num_replicas = 5, gc.ttlseconds = 100000;
```

```text
CONFIGURE ZONE 1
```

```sql
> SHOW ZONE CONFIGURATION FROM DATABASE movr;
```

```text
     target     |              raw_config_sql
----------------+-------------------------------------------
  DATABASE movr | ALTER DATABASE movr CONFIGURE ZONE USING

                |     gc.ttlseconds = 100000,

(1 row)
```

### View the replication zone for a table

To control replication for a specific table, use the `ALTER TABLE... CONFIGURE ZONE` statement to define the relevant values (other values will be inherited from the parent zone):

```sql
> ALTER TABLE users CONFIGURE ZONE USING num_replicas = 5, gc.ttlseconds = 100000;
```

```text
CONFIGURE ZONE 1
```

```sql
> SHOW ZONE CONFIGURATION FROM TABLE users;
```

```text
    target    |             raw_config_sql
--------------+-----------------------------------------
  TABLE users | ALTER TABLE users CONFIGURE ZONE USING
              |     range_min_bytes = 134217728,
              |     range_max_bytes = 536870912,
              |     gc.ttlseconds = 100000,
              |     num_replicas = 5,
              |     constraints = '[]',
              |     lease_preferences = '[]'
(1 row)
```

You can also use [`SHOW CREATE TABLE`](/docs/v23.2/show-create) to view zone configurations for a table. If a table is partitioned, but no zones are configured, the `SHOW CREATE TABLE` output includes a warning.

### View the replication zone for an index

To control replication for a specific index, use the [`ALTER INDEX... CONFIGURE ZONE`](/docs/v23.2/alter-index#configure-zone) statement to define the relevant values (other values will be inherited from the parent zone):

```sql
> ALTER INDEX vehicles@vehicles_auto_index_fk_city_ref_users CONFIGURE ZONE USING num_replicas = 5, gc.ttlseconds = 100000;
```

```text
CONFIGURE ZONE 1
```

```sql
> SHOW ZONE CONFIGURATION FROM INDEX vehicles@vehicles_auto_index_fk_city_ref_users;
```

### View the replication zone for a partition

<Tip>
Most users should not need to use partitioning directly. Instead, they should use CockroachDB's built-in [multi-region capabilities](/docs/v23.2/multiregion-overview), which automatically handle geo-partitioning and other low-level details.
</Tip>

Once [partitions have been defined for a table or a secondary index](/docs/v23.2/partitioning), to control replication for a partition, use `ALTER PARTITION <partition> OF INDEX <table@index> CONFIGURE ZONE`:

```sql
> ALTER PARTITION us_west OF INDEX vehicles@primary
    CONFIGURE ZONE USING
      num_replicas = 5,
      constraints = '[+region=us-west1]';
```

```text
CONFIGURE ZONE 1
```

```sql
> ALTER PARTITION us_west OF INDEX vehicles@vehicles_auto_index_fk_city_ref_users
    CONFIGURE ZONE USING
      num_replicas = 5,
      constraints = '[+region=us-west1]';
```

```text
CONFIGURE ZONE 1
```

To define replication zones for identically named partitions of a table and its secondary indexes, you can use the `<table>@*` syntax to save several steps:

```sql
> ALTER PARTITION us_west OF INDEX vehicles@*
    CONFIGURE ZONE USING
      num_replicas = 5,
      constraints = '[+region=us-west1]';
```

To view the zone configuration for a partition, use `SHOW ZONE CONFIGURATION FROM PARTITION <partition> OF INDEX <table@index>`:

```sql
> SHOW ZONE CONFIGURATION FROM PARTITION us_west OF INDEX vehicles@primary;
```

```text
                    target                    |                             raw_config_sql
----------------------------------------------+-------------------------------------------------------------------------
  PARTITION us_west OF INDEX vehicles@primary | ALTER PARTITION us_west OF INDEX vehicles@primary CONFIGURE ZONE USING
                                              |     range_min_bytes = 134217728,
                                              |     range_max_bytes = 536870912,
                                              |     gc.ttlseconds = 90000,
                                              |     num_replicas = 5,
                                              |     constraints = '[+region=us-west1]',
                                              |     lease_preferences = '[]'
(1 row)
```

<Tip>
You can also use the [`SHOW CREATE TABLE`](/docs/v23.2/show-create) statement or [`SHOW PARTITIONS`](/docs/v23.2/show-partitions) statements to view details about all of the replication zones defined for the partitions of a table and its secondary indexes.
</Tip>
