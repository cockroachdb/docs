This document outlines the foundational skills required to deploy and operate CockroachDB in production environments.

The skills are organized into sections based on the following operational domains:

- [Infrastructure configuration](#infrastructure-configuration)
- [Security](#security)
- [Cluster maintenance](#cluster-maintenance)
- [Troubleshooting](#troubleshooting)
- [Disaster recovery](#disaster-recovery)

Each section includes links to relevant documentation for the listed skills.

Cockroach Labs offers [Professional Services](https://www.cockroachlabs.com/company/professional-services) that can assist you with getting applications into production faster and more efficiently.
</Tip>

## Cluster maintenance

This section covers how to manage the lifecycle of CockroachDB nodes, including adding and removing nodes, handling outages, performing upgrades or downgrades, and modifying cluster settings.

- [Shut down a node gracefully](/docs/v23.2/node-shutdown)
- [Handle unplanned node outages](/docs/v23.2/recommended-production-settings#load-balancing)
- [Add nodes](/docs/v23.2/cockroach-start#add-a-node-to-a-cluster)
- [Remove nodes](/docs/v23.2/node-shutdown?filters=decommission#remove-nodes)
- [Add a region](/docs/v23.2/alter-database#add-regions-to-a-database)
- [Remove a region](/docs/v23.2/alter-database#drop-region)
- [Rolling upgrades](/docs/v23.2/upgrade-cockroach-version)
- Downgrade a cluster from a [patch or major version](/docs/v23.2/upgrade-cockroach-version#step-5-roll-back-the-upgrade-optional)
- [Change a cluster setting](/docs/v23.2/cluster-settings#change-a-cluster-setting)
- Repave a cluster: cluster repaving involves the following individual skills, which are also used during [rolling upgrades](/docs/v23.2/upgrade-cockroach-version):
    1. [Shut down a node gracefully](/docs/v23.2/node-shutdown)
    2. Detach the [persistent volume](/docs/v23.2/kubernetes-overview#kubernetes-terminology) (a.k.a. persistent disk) from the removed node's virtual machine (VM) (this step is optional but recommended)
    3. Delete the removed node's VM
    4. Start a new VM
    5. Reattach the persistent disk to the new VM (necessary if you did step #2)
    6. [Add a node to the cluster](/docs/v23.2/cockroach-start#add-a-node-to-a-cluster) from the new VM
