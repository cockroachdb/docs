The `CREATE VIEW` statement creates a new [view](/docs/v23.2/views), which is a stored query represented as a virtual table.

<Note>
By default, views created in a database cannot reference objects in a different database. To enable cross-database references for views, set the `sql.cross_db_views.enabled` [cluster setting](/docs/v23.2/cluster-settings) to `true`.
</Note>

<Note>
The `CREATE VIEW` statement performs a schema change. For more information about how online schema changes work in CockroachDB, see [Online Schema Changes](/docs/v23.2/online-schema-changes).
</Note>

## Required privileges

The user must have the `CREATE` [privilege](/docs/v23.2/security-reference/authorization#managing-privileges) on the parent database and the `SELECT` privilege on any table(s) referenced by the view.

## Example

<Tip>
This example highlights one key benefit to using views: simplifying complex queries. For additional benefits and examples, see [Views](/docs/v23.2/views).
</Tip>

### Setup

The following examples use the [`startrek` demo database schema](/docs/v23.2/cockroach-demo#datasets).

To follow along, run [`cockroach demo startrek`](/docs/v23.2/cockroach-demo) to start a temporary, in-memory cluster with the `startrek` schema and dataset preloaded:

```shell
$ cockroach demo startrek
```

### Create a view

The sample `startrek` database contains two tables, `episodes` and `quotes`. The table also contains a foreign key constraint, between the `episodes.id` column and the `quotes.episode` column. To count the number of famous quotes per season, you could run the following join:

```sql
> SELECT startrek.episodes.season, count(*)
  FROM startrek.quotes
  JOIN startrek.episodes
  ON startrek.quotes.episode = startrek.episodes.id
  GROUP BY startrek.episodes.season;
```

```text
  season | count
---------+--------
       1 |    78
       2 |    76
       3 |    46
(3 rows)
```

Alternatively, to make it much easier to run this complex query, you could create a view:

```sql
> CREATE VIEW startrek.quotes_per_season (season, quotes)
  AS SELECT startrek.episodes.season, count(*)
  FROM startrek.quotes
  JOIN startrek.episodes
  ON startrek.quotes.episode = startrek.episodes.id
  GROUP BY startrek.episodes.season;
```

The view is then represented as a virtual table alongside other tables in the database:

```sql
> SHOW TABLES FROM startrek;
```

```text
  schema_name |    table_name     | type  | estimated_row_count
--------------+-------------------+-------+----------------------
  public      | episodes          | table |                  79
  public      | quotes            | table |                 200
  public      | quotes_per_season | view  |                   3
(3 rows)
```

Executing the query is as easy as `SELECT`ing from the view, as you would from a standard table:

```sql
> SELECT * FROM startrek.quotes_per_season;
```

```text
  season | quotes
---------+---------
       3 |     46
       1 |     78
       2 |     76
(3 rows)
```

### Replace an existing view

You can create a new view, or replace an existing view, with `CREATE OR REPLACE VIEW`:

```sql
> CREATE OR REPLACE VIEW startrek.quotes_per_season (season, quotes)
  AS SELECT startrek.episodes.season, count(*)
  FROM startrek.quotes
  JOIN startrek.episodes
  ON startrek.quotes.episode = startrek.episodes.id
  GROUP BY startrek.episodes.season
  ORDER BY startrek.episodes.season;
```

```sql
> SELECT * FROM startrek.quotes_per_season;
```

```text
  season | quotes
---------+---------
       3 |     46
       1 |     78
       2 |     76
(3 rows)
```
