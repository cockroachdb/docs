The `DROP ROLE` [statement](/docs/v23.2/sql-statements) removes one or more SQL roles. You can use the keywords `ROLE` and `USER` interchangeably. [`DROP USER`](/docs/v23.2/drop-user) is an alias for `DROP ROLE`.

<Note>
The `DROP ROLE` statement performs a schema change. For more information about how online schema changes work in CockroachDB, see [Online Schema Changes](/docs/v23.2/online-schema-changes).
</Note>

## Considerations

- The `admin` user/role cannot be dropped, and `root` must always be a member of `admin`.
- A user/role cannot be dropped if it has privileges. Use [`REVOKE`](/docs/v23.2/revoke) to remove privileges.
- Users/roles that [own objects](/docs/v23.2/security-reference/authorization#object-ownership) (such as databases, tables, schemas, and types) cannot be dropped until the [ownership is transferred to another user/role](/docs/v23.2/alter-database#change-a-databases-owner).
- If a user/role is logged in while a [different session](/docs/v23.2/show-sessions) drops that user, CockroachDB checks that the user exists before allowing it to inherit privileges from [the `public` role](/docs/v23.2/security-reference/authorization). In addition, any active [web](/docs/v23.2/ui-overview#authentication) [sessions](/docs/v23.2/cockroach-auth-session) are revoked when a user is dropped.

## Required privileges

Non-admin roles cannot drop admin roles. To drop non-admin roles, the role must be a member of the `admin` role or have the [`CREATEROLE`](/docs/v23.2/create-role#create-a-role-that-can-create-other-roles-and-manage-authentication-methods-for-the-new-roles) parameter set.

## Example

In this example, first check a role's privileges. Then, revoke the role's privileges and remove the role.

```sql
> SHOW GRANTS ON documents FOR dev_ops;
```

```text
+------------+--------+-----------+---------+------------+
|  Database  | Schema |   Table   |  User   | Privileges |
+------------+--------+-----------+---------+------------+
| jsonb_test | public | documents | dev_ops | INSERT     |
+------------+--------+-----------+---------+------------+
```

```sql
> REVOKE INSERT ON documents FROM dev_ops;
```

<Note>
All of a role's privileges must be revoked before the role can be dropped.
</Note>

```sql
> DROP ROLE dev_ops;
```
