The `SHOW ROLES` [statement](/docs/v23.2/sql-statements) lists the roles for all databases.

<Note>
Since the keywords `ROLES` and `USERS` can now be used interchangeably in SQL statements for enhanced PostgreSQL compatibility, `SHOW ROLES` is now an alias for [`SHOW USERS`](/docs/v23.2/show-users).
</Note>

## Required privileges

The role must have the [`SELECT`](/docs/v23.2/select-clause) [privilege](/docs/v23.2/security-reference/authorization#managing-privileges) on the `system.users` and `system.role_members` tables.

## Example

```sql
> SHOW ROLES;
```

```text
  username |  options   | member_of
-----------+------------+------------
  admin    | CREATEROLE | {}
  carl     | NOLOGIN    | {}
  petee    |            | {}
  root     | CREATEROLE | {admin}
(4 rows)
```
