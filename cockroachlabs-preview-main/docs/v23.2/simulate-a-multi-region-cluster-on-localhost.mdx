Once you've [installed CockroachDB](/docs/v23.2/install-cockroachdb), you can simulate multi-region cluster on your local machine using [`cockroach demo`](/docs/v23.2/cockroach-demo)to learn about CockroachDB's [multi-region abstractions](/docs/v23.2/multiregion-overview).

<Note>
[`cockroach demo`](/docs/v23.2/cockroach-demo) is not suitable for production deployments. Additionally, simulating multiple geographically distributed nodes on a single host is not representative of the [performance you should expect](/docs/v23.2/frequently-asked-questions#single-row-perf) in a production deployment. To learn more about production multi-region deployments, refer to [Orchestrate CockroachDB Across Multiple Kubernetes Clusters](/docs/v23.2/orchestrate-cockroachdb-with-kubernetes-multi-cluster) and [Deploy a Global, Serverless Application](/docs/v23.2/movr-flask-deployment), and review the [Production Checklist](/docs/v23.2/recommended-production-settings).
</Note>

## Before you begin

[Download](/docs/releases) and [Install CockroachDB](/docs/v23.2/install-cockroachdb).

## Step 1. Start the cluster

Use the following [`cockroach demo`](/docs/v23.2/cockroach-demo) command to start the cluster. This particular combination of flags results in a demo cluster of 9 nodes, with 3 nodes in each region. It sets the appropriate [node localities](/docs/v23.2/cockroach-start#locality) and also simulates the network latency that would occur between nodes in these localities. For more information about each flag, see the [`cockroach demo`](/docs/v23.2/cockroach-demo#flags) documentation, especially for [`--global`](/docs/v23.2/cockroach-demo#global-flag).

```shell
$ cockroach demo --global --nodes 9 --no-example-database --insecure
```

When the cluster starts, you'll see a message like the one shown below, followed by a SQL prompt. Note the URLs for:

- Viewing the [DB Console](/docs/v23.2/ui-overview): `http://127.0.0.1:8080`.
- Connecting to the database from a [SQL shell](/docs/v23.2/cockroach-sql) or a [programming language](/docs/v23.2/connect-to-the-database): `postgres://root@127.0.0.1:26257?sslmode=disable`.

## Step 2. Enter SQL statements at the prompt

Now that your simulated multi-region cluster is running, you are presented with a SQL prompt:

```text
root@127.0.0.1:26257/defaultdb>
```

You can run some basic [CockroachDB SQL statements](/docs/v23.2/learn-cockroachdb-sql):

```sql
> CREATE DATABASE bank;
```

```sql
> USE bank;
```

```sql
> CREATE TABLE bank.accounts (id INT PRIMARY KEY, balance DECIMAL);
```

```sql
> INSERT INTO bank.accounts VALUES (1, 1000.50);
```

```sql
> SELECT * FROM bank.accounts;
```

```text
  id | balance
+----+---------+
   1 | 1000.50
(1 row)
```

## Step 3. View cluster health and performance in the DB Console

The [DB Console](/docs/v23.2/ui-overview) gives you insight into the overall health of your cluster, as well as the performance of any [client workloads](/docs/v23.2/cockroach-workload).

To verify that the 9 nodes you specified are up and running as expected, go to the [**Cluster Overview**](/docs/v23.2/ui-cluster-overview-page) at [http://localhost:8080/#/overview/list](http://localhost:8080/#/overview/list).

<Note>
Capacity metrics can be incorrect when running multiple nodes on a single machine. For more details, see this [limitation](/docs/v23.2/known-limitations#available-capacity-metric-in-the-db-console).
</Note>

To see which nodes are located in which regions, and to see the simulated latencies between them, go to the [**Network Diagnostics**](/docs/v23.2/ui-network-latency-page) page at [http://localhost:8080/#/reports/network/region](http://localhost:8080/#/reports/network/region)

To access a variety of time series charts, including graphs of [SQL queries](/docs/v23.2/ui-sql-dashboard#sql-statements) and [SQL service latency](/docs/v23.2/ui-sql-dashboard#service-latency-sql-99th-percentile), go to the [**Metrics**](/docs/v23.2/ui-overview-dashboard) page at [http://localhost:8080/#/metrics](http://localhost:8080/#/metrics)

You may also find the following pages useful:

- The [**Databases**](/docs/v23.2/ui-databases-page) page at [http://localhost:8080/#/databases](http://localhost:8080/#/databases) shows details about your databases and tables.
- The [**Statements**](/docs/v23.2/ui-statements-page) page at [http://localhost:8080/#/statements](http://localhost:8080/#/statements) is used to assess the performance of specific queries.
- The [**Jobs**](/docs/v23.2/ui-jobs-page) page at [http://localhost:8080/#/jobs](http://localhost:8080/#/jobs) is used to monitor the status of long-running operations like [schema changes](/docs/v23.2/online-schema-changes).

## Step 4. Wipe the cluster

When you're done with your demo cluster, you can wipe the cluster by typing the command shown below at the SQL prompt. Note that because this is a temporary, in-memory cluster, this will wipe any data you have added to the cluster. For more information, see the [`cockroach demo`](/docs/v23.2/cockroach-demo) documentation.

```sql
\quit
```
