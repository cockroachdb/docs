The `SHOW STATISTICS` [statement](/docs/v25.2/sql-statements) lists [table statistics](/docs/v25.2/create-statistics) used by the [cost-based optimizer](/docs/v25.2/cost-based-optimizer).

By default, CockroachDB [automatically generates statistics](/docs/v25.2/cost-based-optimizer#table-statistics) on all indexed columns and up to 100 non-indexed columns, and automatically collects [multi-column statistics](/docs/v25.2/create-statistics#create-statistics-on-multiple-columns) on the columns that prefix each index.

## Synopsis

<ShowStatistics5 />

## Required Privileges

To list table statistics, the user must have any [privilege](/docs/v25.2/security-reference/authorization#managing-privileges) on the table being inspected.

## Parameters

<table><thead><tr><th>Parameter</th><th>Description</th></tr></thead><tbody><tr><td><code>table&#95;name</code></td><td>The name of the table to view statistics for.</td></tr><tr><td><code>opt&#95;with&#95;options</code></td><td>Control the behavior of <code>SHOW STATISTICS</code> with <a href="#options">these options</a>.</td></tr></tbody></table>

### Options

<table><thead><tr><th>Option</th><th>Value</th><th>Description</th></tr></thead><tbody><tr><td><code>FORECAST</code></td><td>N/A</td><td><a href="#display-forecasted-statistics">Display forecasted statistics</a> along with the existing table statistics.</td></tr></tbody></table>

## Output

<table><thead><tr><th>Column</th><th>Description</th></tr></thead><tbody><tr><td><code>statistics&#95;name</code></td><td>The name of the statistics. If <code>&#95;&#95;auto&#95;&#95;</code>, the statistics were created automatically. If <code>&#95;&#95;forecast&#95;&#95;</code>, the statistics are <a href="#display-forecasted-statistics">forecasted</a>.</td></tr><tr><td><code>column&#95;names</code></td><td>The name of the columns on which the statistics were created.</td></tr><tr><td><code>created</code></td><td>The <a href="/docs/v25.2/timestamp"><code>timestamptz</code></a> when the statistics were created.</td></tr><tr><td><code>row&#95;count</code></td><td>The number of rows for which the statistics were computed.</td></tr><tr><td><code>distinct&#95;count</code></td><td>The number of distinct values for which the statistics were computed.</td></tr><tr><td><code>null&#95;count</code></td><td>The number of null values for which the statistics were computed.</td></tr><tr><td><code>avg&#95;size</code></td><td>The average size in bytes of the values of the columns for which the statistics were computed.</td></tr><tr><td><code>histogram&#95;id</code></td><td>The ID of the <a href="/docs/v25.2/cost-based-optimizer#control-histogram-collection">histogram</a> used to compute statistics.</td></tr></tbody></table>

## Examples

#### Setup

To follow along, run [`cockroach demo`](/docs/v25.2/cockroach-demo) to start a temporary, in-memory cluster with the [`movr`](/docs/v25.2/movr) sample dataset preloaded:

```shell
$ cockroach demo
```

### Delete statistics

To delete statistics for all tables in all databases:

```sql
DELETE FROM system.table_statistics WHERE true;
```

To delete a named set of statistics (e.g, one named "users_stats"), run a query like the following:

```sql
DELETE FROM system.table_statistics WHERE name = 'users_stats';
```

For more information about the `DELETE` statement, see [`DELETE`](/docs/v25.2/delete).
