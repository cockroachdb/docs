The `ALTER PROCEDURE` [statement](/docs/v25.2/sql-statements) modifies a [stored procedure](/docs/v25.2/stored-procedures).

### `OWNER TO`

`ALTER PROCEDURE... OWNER TO` is used to change the owner of a procedure.

#### Required privileges

- To alter the owner of a procedure, the new owner must have [`CREATE` privilege](/docs/v25.2/security-reference/authorization#supported-privileges) on the schema of the procedure.
- To alter a procedure, a user must [own](/docs/v25.2/security-reference/authorization#object-ownership) the procedure.
- To alter a procedure, a user must have [`DROP` privilege](/docs/v25.2/security-reference/authorization#supported-privileges) on the schema of the procedure.

#### Parameters

<table><thead><tr><th>Parameter</th><th>Description</th></tr></thead><tbody><tr><td><code>role&#95;spec</code></td><td>The role to set as the owner of the procedure.</td></tr></tbody></table>

See [Synopsis](#synopsis).

### `RENAME TO`

`ALTER PROCEDURE... RENAME TO` changes the name of a procedure.

#### Required privileges

- To alter a procedure, a user must [own](/docs/v25.2/security-reference/authorization#object-ownership) the procedure.
- To alter a procedure, a user must have [`DROP` privilege](/docs/v25.2/security-reference/authorization#supported-privileges) on the schema of the procedure.

#### Parameters

<table><thead><tr><th>Parameter</th><th>Description</th></tr></thead><tbody><tr><td><code>proc&#95;new&#95;name</code></td><td>The new name of the procedure.</td></tr></tbody></table>

See [Synopsis](#synopsis).

### `SET SCHEMA`

`ALTER PROCEDURE... SET SCHEMA` changes the [schema](/docs/v25.2/sql-name-resolution) of a procedure.

<Note>
CockroachDB supports `SET SCHEMA` as an [alias for setting the `search_path` session variable](/docs/v25.2/set-vars#supported-variables).
</Note>

#### Required privileges

- To change the schema of a procedure, a user must have [`CREATE` privilege](/docs/v25.2/security-reference/authorization#supported-privileges) on the new schema.
- To alter a procedure, a user must [own](/docs/v25.2/security-reference/authorization#object-ownership) the procedure.
- To alter a procedure, a user must have [`DROP` privilege](/docs/v25.2/security-reference/authorization#supported-privileges) on the schema of the procedure.

#### Parameters

<table><thead><tr><th>Parameter</th><th>Description</th></tr></thead><tbody><tr><td><code>schema&#95;name</code></td><td>The name of the new schema for the procedure.</td></tr></tbody></table>

See [Synopsis](#synopsis).

## Examples

### Rename a stored procedure

The following statement renames the [`delete_earliest_histories` example procedure](/docs/v25.2/stored-procedures#create-a-stored-procedure-using-pl-pgsql) to `delete_histories`:

```sql
ALTER PROCEDURE delete_earliest_histories RENAME TO delete_histories;
```
