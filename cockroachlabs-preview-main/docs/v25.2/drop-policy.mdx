The `DROP POLICY` statement removes an existing [row-level security (RLS)](/docs/v25.2/enum) policy from a [table](/docs/v25.2/schema-design-table).

## Syntax

```text
DROP POLICY [ IF EXISTS ] policy_name ON table_name [ CASCADE | RESTRICT ];
```

## Parameters

<table><thead><tr><th>Parameter</th><th>Description</th></tr></thead><tbody><tr><td><code>policy&#95;name</code></td><td>Unique identifier for the policy on the table.</td></tr><tr><td><code>table&#95;name</code></td><td>The <a href="/docs/v25.2/schema-design-table">table</a> to which the policy applies.</td></tr><tr><td><code>IF EXISTS</code></td><td>Suppresses an error if the policy doesn't exist.</td></tr><tr><td><code>CASCADE, RESTRICT</code></td><td>Standard dependency handling (not relevant for policies themselves).</td></tr></tbody></table>

## Examples

### Drop a policy

To drop an existing policy, issue the following statement:

```sql
DROP POLICY IF EXISTS your_policy ON orders;
```
