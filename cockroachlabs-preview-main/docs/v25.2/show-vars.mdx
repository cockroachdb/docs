Use the `SHOW` [statement](/docs/v25.2/sql-statements) to display the value of one or all of the session variables. You configure session variables using [`SET`](/docs/v25.2/set-vars).

## Required privileges

No [privileges](/docs/v25.2/security-reference/authorization#managing-privileges) are required to display the session variables.

## Synopsis

<ShowVars5 />

<Note>
The `SHOW` statement for session variables is unrelated to the other `SHOW` statements like [`SHOW CLUSTER SETTING`](/docs/v25.2/show-cluster-setting), [`SHOW CREATE`](/docs/v25.2/show-create), and [`SHOW DATABASES`](/docs/v25.2/show-databases).
</Note>

### Showing the value of a single session variable

```sql
> SHOW DATABASE;
```

```text
------------
  movr
(1 row)
```

```sql
> SHOW ALL;
```

```text
       variable       |      value
----------------------+-------------------
  application_name    | $ cockroach demo
  bytea_output        | hex
  client_encoding     | UTF8
  client_min_messages | notice
...
```
