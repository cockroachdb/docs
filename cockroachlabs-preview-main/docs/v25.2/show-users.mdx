The `SHOW USERS` [statement](/docs/v25.2/sql-statements) lists the users for all databases.

<Note>
Since the keywords `ROLES` and `USERS` can now be used interchangeably in SQL statements for enhanced PostgreSQL compatibility, `SHOW USERS` is now an alias for [`SHOW ROLES`](/docs/v25.2/show-roles).
</Note>

## Required privileges

The user must have the [`SELECT`](/docs/v25.2/select-clause) [privilege](/docs/v25.2/security-reference/authorization#managing-privileges) on the `system.users` and `system.role_members` tables.

## Example

```sql
> SHOW USERS;
```

```text
  username |  options   | member_of
-----------+------------+------------
  admin    | CREATEROLE | {}
  carl     | NOLOGIN    | {}
  petee    |            | {}
  root     | CREATEROLE | {admin}
(4 rows)
```

Alternatively, within the built-in SQL shell, you can use the `\du` [shell command](/docs/v25.2/cockroach-sql#commands):

```sql
> \du
```

```text
  username |  options   | member_of
-----------+------------+------------
  admin    | CREATEROLE | {}
  carl     | NOLOGIN    | {}
  petee    |            | {}
  root     | CREATEROLE | {admin}
(4 rows)
```
