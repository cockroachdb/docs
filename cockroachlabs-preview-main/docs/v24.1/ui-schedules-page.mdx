The **Schedules** page of the DB Console provides details about the scheduled tasks performed by your cluster. These can include:

- [Scheduled backups](/docs/v24.1/manage-a-backup-schedule)
- [Scheduled auto statistics compaction](/docs/v24.1/cost-based-optimizer#table-statistics)
- [Row-level TTL](/docs/v24.1/row-level-ttl)

To view these details, [access the DB console](/docs/v24.1/ui-overview#db-console-access) and click **Schedules** in the left-hand navigation.

## Filter schedules

- Use the **Status** menu to filter schedules by schedule status, displaying all schedules, only active schedules, or only paused schedules.
- Use the **Show** menu to toggle displaying the latest 50 schedules or all schedules on the cluster.

## Schedules list

Use the **Schedules** list to see your active and paused schedules.

- To view schedule details click the schedule ID.
- If you drop a schedule, it will no longer be listed.

## Schedule details

Click on a schedule ID to view the full SQL statement that the schedule runs. For example, the following screenshot shows the resulting [`BACKUP`](/docs/v24.1/backup) statement for a full cluster backup recurring every day:

  ![UI for each individual schedule ID displaying the CREATE SCHEDULE SQL statement](/images/v24.1/schedule-id-screen.png)

You may also view a `protected_timestamp_record` on this page. This indicates that the schedule is actively managing its own [protected timestamp](/docs/v24.1/architecture/storage-layer#protected-timestamps) records independently of [GC TTL](/docs/v24.1/configure-replication-zones#gc-ttlseconds). See [Protected timestamps and scheduled backups](/docs/v24.1/create-schedule-for-backup#protected-timestamps-and-scheduled-backups) for more detail.
