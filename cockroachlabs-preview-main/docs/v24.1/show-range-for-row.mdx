The `SHOW RANGE... FOR ROW` [statement](/docs/v24.1/sql-statements) shows information about a [range](/docs/v24.1/architecture/overview#architecture-range) for a single row in a table or index. This information is useful for verifying how SQL data maps to underlying ranges, and where the replicas for a range are located.

<Note>
**This feature is in [preview](/docs/v24.1/cockroachdb-feature-availability)** and subject to change. To share feedback and/or issues, contact [Support](https://support.cockroachlabs.com/hc/en-us).
</Note>

<Note>
To show information about the ranges for all data in a table, index, or database, use the [`SHOW RANGES`](/docs/v24.1/show-ranges) statement.
</Note>

## Syntax

```text
SHOW RANGE FROM TABLE <tablename> FOR ROW (value1, value2, ...)
SHOW RANGE FROM INDEX [ <tablename> @ ] <indexname> FOR ROW (value1, value2, ...)
```

## Required privileges

The user must have the `SELECT` [privilege](/docs/v24.1/security-reference/authorization#managing-privileges) on the target table.

## Parameters

<table><thead><tr><th>Parameter</th><th>Description</th></tr></thead><tbody><tr><td><code>tablename</code></td><td>The name of the table that contains the row that you want range information about.</td></tr><tr><td><code>indexname</code></td><td>The name of the index that contains the row that you want range information about.</td></tr><tr><td><code>(value1, value2, ...)</code></td><td>The values of the indexed columns of the row that you want range information about, as a tuple. In previous releases, this statement required the values of all columns of a row.</td></tr></tbody></table>

## Response

The following fields are returned:

<table><thead><tr><th>Field</th><th>Description</th></tr></thead><tbody><tr><td><code>start&#95;key</code></td><td>The start key for the range.</td></tr><tr><td><code>end&#95;key</code></td><td>The end key for the range.</td></tr><tr><td><code>range&#95;id</code></td><td>The range ID.</td></tr><tr><td><code>lease&#95;holder</code></td><td>The node that contains the range's <a href="/docs/v24.1/architecture/overview#architecture-range">leaseholder</a>.</td></tr><tr><td><code>lease&#95;holder&#95;locality</code></td><td>The <a href="/docs/v24.1/cockroach-start#locality">locality</a> of the leaseholder.</td></tr><tr><td><code>replicas</code></td><td>The nodes that contain the range <a href="/docs/v24.1/architecture/overview#architecture-range">replicas</a>.</td></tr><tr><td><code>replica&#95;localities</code></td><td>The <a href="/docs/v24.1/cockroach-start#locality">locality</a> of the range.</td></tr></tbody></table>

## Examples

#### Setup

To follow along, run [`cockroach demo`](/docs/v24.1/cockroach-demo) to start a temporary, in-memory cluster with the [`movr`](/docs/v24.1/movr) sample dataset preloaded:

```shell
$ cockroach demo
```
