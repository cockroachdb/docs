The `RESET` [statement](/docs/v24.1/sql-statements) resets a [cluster setting](/docs/v24.1/set-cluster-setting) to its default value for the client session.

## Required privileges

Only members of the `admin` role can modify cluster settings. By default, the `root` user belongs to the `admin` role.

## Example

<Tip>
You can use [`SET CLUSTER SETTING.. TO DEFAULT`](/docs/v24.1/set-cluster-setting) to reset a cluster setting as well.
</Tip>

```sql
> SET CLUSTER SETTING sql.metrics.statement_details.enabled = false;
```

```sql
> SHOW CLUSTER SETTING sql.metrics.statement_details.enabled;
```

```text
  sql.metrics.statement_details.enabled
-----------------------------------------
                  false
(1 row)
```

```sql
> RESET CLUSTER SETTING sql.metrics.statement_details.enabled;
```

```sql
> SHOW CLUSTER SETTING sql.metrics.statement_details.enabled;
```

```text
  sql.metrics.statement_details.enabled
-----------------------------------------
                  true
(1 row)
```
