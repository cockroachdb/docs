This page documents best practices for unit testing applications built on CockroachDB in a local environment.

If you are deploying a self-hosted cluster, see the [Production Checklist](/docs/v24.1/recommended-production-settings) for information about preparing your cluster for production.

<Warning>
The settings described on this page are **not recommended** for use in production clusters. They are only recommended for use during unit testing and continuous integration testing (CI).
</Warning>

## Use a local, single-node cluster with in-memory storage

The [`cockroach start-single-node`](/docs/v24.1/cockroach-start-single-node) command below starts a single-node, insecure cluster with [in-memory storage](/docs/v24.1/cockroach-start-single-node#store). Using in-memory storage improves the speed of the cluster for local testing purposes.

```shell
cockroach start-single-node --insecure --store=type=mem,size=0.25 --advertise-addr=localhost
```

We recommend the following additional [cluster settings](/docs/v24.1/cluster-settings) and [SQL statements](/docs/v24.1/sql-statements#data-definition-statements) for improved performance during functional unit testing and continuous integration testing. In particular, some of these settings will increase the performance of [schema changes](/docs/v24.1/online-schema-changes), since repeated [creation](/docs/v24.1/create-schema) and [dropping](/docs/v24.1/drop-schema) of schemas are common in automated testing.

To change all of the settings described above at once, run the following SQL statements:

```sql
SET CLUSTER SETTING kv.range_merge.queue_interval = '50ms';
SET CLUSTER SETTING jobs.registry.interval.gc = '30s';
SET CLUSTER SETTING jobs.registry.interval.cancel = '180s';
SET CLUSTER SETTING jobs.retention_time = '15s';
SET CLUSTER SETTING sql.stats.automatic_collection.enabled = false;
SET CLUSTER SETTING kv.range_split.by_load_merge_delay = '5s';
ALTER RANGE default CONFIGURE ZONE USING "gc.ttlseconds" = 600;
ALTER DATABASE system CONFIGURE ZONE USING "gc.ttlseconds" = 600;
```

<Warning>
These settings **are not** recommended for [performance benchmarking of CockroachDB](/docs/v24.1/performance-benchmarking-with-tpcc-local) since they will lead to inaccurate results.
</Warning>

## Scope tests to a database when possible

It is better to scope tests to a [database](/docs/v24.1/create-database) than to a [user-defined schema](/docs/v24.1/create-schema) due to inefficient user-defined schema validation. The performance of user-defined schema validation may be improved in a future release.

## Log test output to a file

By default, `cockroach start-single-node` logs cluster activity to a file with the [default logging configuration](/docs/v24.1/configure-logs#default-logging-configuration). When you specify the `--store=type=mem` flag, the command prints cluster activity directly to the console instead.

To customize logging behavior for local clusters, use the [`--log` flag](/docs/v24.1/cockroach-start-single-node#logging):

```shell
cockroach start-single-node --insecure --store=type=mem,size=0.25 --advertise-addr=localhost --log="{file-defaults: {dir: /path/to/logs}, sinks: {stderr: {filter: NONE}}}"
```

The `log` flag has two suboptions:

- `file-defaults`, which specifies the path of the file in which to log events ( `/path/to/logs` ).
- `sinks`, which provides a secondary destination to which to log events ( `stderr` ).

For more information about logging, see [Configure logs](/docs/v24.1/configure-logs).

## Use a local file server for bulk operations

To test bulk operations like [`BACKUP`](/docs/v24.1/backup) or [`RESTORE`](/docs/v24.1/restore), we recommend using a local file server.

For more details, see [Use a Local File Server](/docs/v24.1/use-a-local-file-server).

## Use Docker-specific testing and development tools

When you use the `cockroach start-single-node` command to start a single-node cluster with Docker, some additional features are available to help with testing and development. Refer to [Start a local cluster in Docker (Linux)](/docs/v24.1/start-a-local-cluster-in-docker) and [Start a local cluster in Docker (macOS)](/docs/v24.1/start-a-local-cluster-in-docker).
