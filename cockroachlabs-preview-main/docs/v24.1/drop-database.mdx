The `DROP DATABASE` [statement](/docs/v24.1/sql-statements) removes a database and all its objects from a CockroachDB cluster.

<Note>
The `DROP DATABASE` statement performs a schema change. For more information about how online schema changes work in CockroachDB, see [Online Schema Changes](/docs/v24.1/online-schema-changes).
</Note>

## Required privileges

The user must have the `DROP` [privilege](/docs/v24.1/security-reference/authorization#managing-privileges) on the database and on all tables in the database.

## Viewing schema changes

This schema change statement is registered as a job. You can view long-running jobs with [`SHOW JOBS`](/docs/v24.1/show-jobs).

## Examples

#### Setup

To follow along, run [`cockroach demo`](/docs/v24.1/cockroach-demo) to start a temporary, in-memory cluster with the [`movr`](/docs/v24.1/movr) sample dataset preloaded:

```shell
$ cockroach demo
```

### Drop a database and its objects (`CASCADE`)

For non-interactive sessions (e.g., client applications), `DROP DATABASE` applies the `CASCADE` option by default, which drops all tables and views in the database as well as all objects (such as [constraints](/docs/v24.1/constraints) and [views](/docs/v24.1/views)) that depend on those tables.

For interactive sessions from the [built-in SQL client](/docs/v24.1/cockroach-sql), either the `CASCADE` option must be set explicitly or the `--unsafe-updates` flag must be set when starting the shell.

```sql
> SHOW TABLES FROM movr;
```

```text
  schema_name |         table_name         | type  | estimated_row_count
--------------+----------------------------+-------+----------------------
  public      | promo_codes                | table |                1000
  public      | rides                      | table |                 500
  public      | user_promo_codes           | table |                   0
  public      | users                      | table |                  50
  public      | vehicle_location_histories | table |                1000
  public      | vehicles                   | table |                  15
(6 rows)
```

```sql
> DROP DATABASE movr;
```

```text
ERROR: rejected (sql_safe_updates = true): DROP DATABASE on current database
SQLSTATE: 01000
```

```sql
> USE defaultdb;
```

```sql
> DROP DATABASE movr;
```

```text
ERROR: rejected (sql_safe_updates = true): DROP DATABASE on non-empty database without explicit CASCADE
SQLSTATE: 01000
```

```sql
> DROP DATABASE movr CASCADE;
```

```sql
> SHOW TABLES FROM movr;
```

```text
ERROR: target database or schema does not exist
SQLSTATE: 3F000
```

### Prevent dropping a non-empty database (`RESTRICT`)

When a database is not empty, the `RESTRICT` option prevents the database from being dropped:

```sql
> SHOW TABLES FROM movr;
```

```text
  schema_name |         table_name         | type  | estimated_row_count
--------------+----------------------------+-------+----------------------
  public      | promo_codes                | table |                1000
  public      | rides                      | table |                 500
  public      | user_promo_codes           | table |                   0
  public      | users                      | table |                  50
  public      | vehicle_location_histories | table |                1000
  public      | vehicles                   | table |                  15
(6 rows)
```

```sql
> USE defaultdb;
```

```sql
> DROP DATABASE movr RESTRICT;
```

```text
ERROR: database "movr" is not empty and RESTRICT was specified
SQLSTATE: 2BP01
```
