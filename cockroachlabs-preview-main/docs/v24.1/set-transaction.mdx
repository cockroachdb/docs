The `SET TRANSACTION` [statement](/docs/v24.1/sql-statements) sets the transaction priority, access mode, "as of" timestamp, and isolation level. These are applied after you [`BEGIN`](/docs/v24.1/begin-transaction) the transaction and before executing the first statement that manipulates a database.

<Note>
Cockroach Labs recommends leaving the transaction priority at the default setting in almost all cases. Changing the transaction priority to `HIGH` in particular can lead to difficult-to-debug interactions with other transactions executing on the system.

If you are setting a transaction priority to avoid [contention](/docs/v24.1/performance-best-practices-overview#transaction-contention) or [hot spots](/docs/v24.1/performance-best-practices-overview#hot-spots), or to [get better query performance](/docs/v24.1/make-queries-fast), it is usually a sign that you need to update your [schema design](/docs/v24.1/schema-design-database) and/or review the data access patterns of your workload.
</Note>

## Required privileges

No [privileges](/docs/v24.1/security-reference/authorization#managing-privileges) are required to set the transaction priority. However, privileges are required for each statement within a transaction.

## Examples

### Set isolation level

You can set the transaction isolation level to [`SERIALIZABLE`](/docs/v24.1/demo-serializable) or [`READ COMMITTED`](/docs/v24.1/read-committed#enable-read-committed-isolation).

If not specified, transactions use the value of the current session's [`default_transaction_isolation`](/docs/v24.1/session-variables#default-transaction-isolation) variable.

```sql
BEGIN;
```

```sql
SET TRANSACTION ISOLATION LEVEL READ COMMITTED;
```

### Set priority

<Warning>
This example assumes you're using [client-side retry handling](/docs/v24.1/transaction-retry-error-reference#client-side-retry-handling).
</Warning>

```sql
> BEGIN;
```

```sql
> SET TRANSACTION PRIORITY HIGH;
```

```sql
> SAVEPOINT cockroach_restart;
```

```sql
> UPDATE products SET inventory = 0 WHERE sku = '8675309';
```

```sql
> INSERT INTO orders (customer, sku, status) VALUES (1001, '8675309', 'new');
```

```sql
> RELEASE SAVEPOINT cockroach_restart;
```

```sql
> COMMIT;
```

### Use the `AS OF SYSTEM TIME` option

You can execute the transaction using the database contents "as of" a specified time in the past.

```sql
> BEGIN;
```

```sql
> SET TRANSACTION AS OF SYSTEM TIME '2019-04-09 18:02:52.0+00:00';
```

```sql
> SELECT * FROM orders;
```

```sql
> SELECT * FROM products;
```

```sql
> COMMIT;
```

### Set the default transaction priority for a session

To set the default transaction priority for all transactions in a session, use the `default_transaction_priority` [session variable](/docs/v24.1/set-vars). For example:

```sql
> SET default_transaction_priority 'high';
```

```sql
> SHOW transaction_priority;
```

```text
  transaction_priority
------------------------
  high
```

Note that `transaction_priority` is a read-only [session variable](/docs/v24.1/show-vars) that cannot be set directly.
