CockroachDB is officially integrated with the following third-party monitoring platforms. These integrations enable external tools and interfaces

- to collect, visualize, and alert on CockroachDB [metrics](#metrics) and
- to collect CockroachDB [logs](#logs).

## Monitoring integration availability

### Metrics

<table><thead><tr><th>Platform</th><th>CockroachDB Standard</th><th>CockroachDB Advanced</th><th>CockroachDB self-hosted</th></tr></thead><tbody><tr><td><a href="#amazon-cloudwatch-metrics">Amazon CloudWatch</a></td><td>✔</td><td>✔</td><td>✔</td></tr><tr><td><a href="#datadog">DataDog</a></td><td>✔</td><td>✔</td><td>✔</td></tr><tr><td><a href="#dbmarlin">DBmarlin</a></td><td></td><td></td><td>✔</td></tr><tr><td><a href="#kibana">Kibana</a></td><td></td><td></td><td>✔</td></tr><tr><td><a href="#prometheus">Prometheus</a></td><td>✔</td><td>✔</td><td>✔</td></tr></tbody></table>

This list is not exhaustive. Any Prometheus-compatible third-party tool can consume metrics from a CockroachDB Advanced or self-hosted cluster. For example, because [Amazon CloudWatch is Prometheus-compatible](https://docs.aws.amazon.com/AmazonCloudWatch/latest/monitoring/ContainerInsights-Prometheus-Setup-configure-ECS), it can consume metrics from a CockroachDB self-hosted cluster.

### Logs

<table><thead><tr><th>Platform</th><th>CockroachDB Standard</th><th>CockroachDB Advanced</th><th>CockroachDB self-hosted</th></tr></thead><tbody><tr><td><a href="#amazon-cloudwatch-logs">Amazon CloudWatch</a></td><td>✔</td><td>✔</td><td>✔</td></tr><tr><td><a href="#gcp-logging">GCP Logging</a></td><td>✔</td><td>✔</td><td>✔</td></tr></tbody></table>

This list is not exhaustive. Any third-party tool that can consume logs from [file-based or network sinks](/docs/v26.1/configure-logs#configure-log-sinks) can also consume logs from a CockroachDB self-hosted cluster.
