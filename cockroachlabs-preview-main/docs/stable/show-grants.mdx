The `SHOW GRANTS` [statement](/docs/v26.1/sql-statements) lists one of the following:

- The [roles](/docs/v26.1/security-reference/authorization#sql-users) granted to [users](/docs/v26.1/security-reference/authorization#sql-users) in a cluster.
- The [privileges](/docs/v26.1/security-reference/authorization#managing-privileges) [granted](/docs/v26.1/grant) to [users](/docs/v26.1/security-reference/authorization#sql-users) on [databases](/docs/v26.1/create-database), [user-defined functions](/docs/v26.1/create-function), [schemas](/docs/v26.1/create-schema), [tables](/docs/v26.1/create-table), [user-defined types](/docs/v26.1/enum), or [external connections](/docs/v26.1/create-external-connection).

## Syntax

### Show privilege grants

Use the following syntax to show the privileges granted to users on database objects:

```text
SHOW GRANTS [ON [DATABASE | FUNCTION | SCHEMA | TABLE | TYPE | EXTERNAL CONNECTION] <targets...>] [FOR <users...>]
```

The object-focused form (`ON...`) shows only privileges granted directly on the object. It does not resolve privileges inherited through role membership. The principal-focused form (`FOR...`) includes privileges inherited through role membership for the specified users and can be combined with `ON...` to filter by object. For an example of inherited grants, refer to [Show direct and inherited grants](#show-direct-and-inherited-grants). For inherited privileges, the `grantee` column reports the role that carries the privilege. To inspect role memberships, use [`SHOW GRANTS ON ROLE`](#show-role-grants) or [`SHOW ROLES`](/docs/v26.1/show-roles).

When `DATABASE` is omitted, the schema, tables, and types in the [current database](/docs/v26.1/sql-name-resolution#current-database) are listed.

### Show role grants

Use the following syntax to the show the role grants for users in a cluster.

```text
SHOW GRANTS ON ROLE [<roles...>] [FOR <users...>]
```

## Parameters

<table><thead><tr><th>Parameter</th><th>Description</th></tr></thead><tbody><tr><td><code>targets</code></td><td>A comma-separated list of database, function, schema, table, or user-defined type names.<br/><br/>If the function name is not unique, you must provide the full function signature.<br/><br/>To list the privilege grants for all tables in the current database, you can use <code>SHOW GRANTS ON TABLE &#42;</code>.</td></tr><tr><td><code>users</code></td><td>A comma-separated list of the <a href="/docs/v26.1/security-reference/authorization#sql-users">users</a> whose privileges or roles you want to show.</td></tr><tr><td><code>roles</code></td><td>A comma-separated list of the roles whose grants you want to show.</td></tr></tbody></table>

## Response

### Privilege grants

Results for `SHOW GRANTS ON...` without `FOR` list only direct grants on the object. Results for `SHOW GRANTS... FOR <users>` include privileges inherited through role membership for the specified users, and the `grantee` column can report roles that grant those privileges.

The `SHOW GRANTS ON [DATABASE | FUNCTION | SCHEMA | TABLE | TYPE | EXTERNAL CONNECTION]` statement can return the following fields, depending on the target object specified:

<table><thead><tr><th>Field</th><th>Description</th></tr></thead><tbody><tr><td><code>database&#95;name</code></td><td>The name of the database.</td></tr><tr><td><code>function&#95;name</code></td><td>The name of the user-defined function.</td></tr><tr><td><code>schema&#95;name</code></td><td>The name of the schema.</td></tr><tr><td><code>table&#95;name</code></td><td>The name of the table.</td></tr><tr><td><code>type&#95;name</code></td><td>The name of the user-defined type.</td></tr><tr><td><code>connection&#95;name</code></td><td>The name of the external connection.</td></tr><tr><td><code>grantee</code></td><td>The name of the user or role that the row reports privileges for. When you omit <code>FOR</code>, this is the direct grantee on the object and has the listed <a href="/docs/v26.1/security-reference/authorization#managing-privileges">privilege</a>. When you use <code>FOR</code>, this can be a role that grants inherited privileges to the specified users.</td></tr><tr><td><code>privilege&#95;type</code></td><td>The name of the privilege.</td></tr><tr><td><code>is&#95;grantable</code></td><td><code>TRUE</code> if the grantee has the grant option on the object; <code>FALSE</code> if not.</td></tr></tbody></table>

### Role grants

The `SHOW GRANTS ON ROLE` statement returns the following fields:

<table><thead><tr><th>Field</th><th>Description</th></tr></thead><tbody><tr><td><code>role&#95;name</code></td><td>The name of the role.</td></tr><tr><td><code>member</code></td><td>The users in the role.</td></tr><tr><td><code>is&#95;admin</code></td><td>If <code>true</code>, the role is an <a href="/docs/v26.1/security-reference/authorization#role-admin">admin</a> role.</td></tr></tbody></table>

## Required privileges

- No [privileges](/docs/v26.1/security-reference/authorization#managing-privileges) are required to view privileges granted to users.
- For `SHOW GRANTS ON ROLES`, the user must have the [`SELECT`](/docs/v26.1/select-clause) [privilege](/docs/v26.1/security-reference/authorization#managing-privileges) on the system table.

### Show direct and inherited grants

In this example, a role has `ALL` on a table and a user inherits that role. The object-focused form (`SHOW GRANTS ON...`) lists only the direct role grant, while the principal-focused form (`SHOW GRANTS... FOR <user>`) lists the inherited privileges for the user.

```sql
CREATE ROLE analysts;
GRANT ALL ON TABLE users TO analysts;
CREATE USER priya;
GRANT analysts TO priya;
```

```sql
SHOW GRANTS ON TABLE users;
```

```text
  database_name | schema_name | table_name | grantee  | privilege_type | is_grantable
----------------+-------------+------------+----------+----------------+---------------
  movr          | public      | users      | admin    | ALL            |      t
  movr          | public      | users      | analysts | ALL            |      f
  movr          | public      | users      | root     | ALL            |      t
(3 rows)
```

```sql
SHOW GRANTS ON TABLE users FOR priya;
```

```text
  database_name | schema_name | table_name | grantee  | privilege_type | is_grantable
----------------+-------------+------------+----------+----------------+---------------
  movr          | public      | users      | analysts | ALL            |      f
(1 row)
```
