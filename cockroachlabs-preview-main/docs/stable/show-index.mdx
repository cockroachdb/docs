The `SHOW INDEX` [statement](/docs/v26.1/sql-statements) returns index information for a table or database.

## Required privileges

The user must have any [privilege](/docs/v26.1/security-reference/authorization#managing-privileges) on the target table or database.

## Aliases

In CockroachDB, the following are aliases for `SHOW INDEX`:

- `SHOW INDEXES`
- `SHOW KEYS`

## Parameters

<table><thead><tr><th>Parameter</th><th>Description</th></tr></thead><tbody><tr><td><code>table&#95;name</code></td><td>The name of the table for which to show indexes.</td></tr><tr><td><code>database&#95;name</code></td><td>The name of the database for which to show indexes.</td></tr></tbody></table>

## Response

The following fields are returned for each column in each index.

<table><thead><tr><th>Field</th><th>Description</th></tr></thead><tbody><tr><td><code>table&#95;name</code></td><td>The name of the table.</td></tr><tr><td><code>index&#95;name</code></td><td>The name of the index.</td></tr><tr><td><code>non&#95;unique</code></td><td>Whether values in the indexed column are unique. Possible values: <code>true</code> or <code>false</code>.</td></tr><tr><td><code>seq&#95;in&#95;index</code></td><td>The position of the column in the index, starting with <code>1</code>.</td></tr><tr><td><code>column&#95;name</code></td><td>The indexed column.</td></tr><tr><td><code>direction</code></td><td>How the column is sorted in the index. Possible values: <code>ASC</code> or <code>DESC</code> for indexed columns; <code>N/A</code> for stored columns.</td></tr><tr><td><code>storing</code></td><td>Whether the <code>STORING</code> clause was used to index the column during <a href="/docs/v26.1/create-index">index creation</a>. Possible values: <code>true</code> or <code>false</code>.</td></tr><tr><td><code>implicit</code></td><td>Whether the column is part of the index despite not being explicitly included during <a href="/docs/v26.1/create-index">index creation</a>. Possible values: <code>true</code> or <code>false</code><br/><br/><a href="/docs/v26.1/primary-key">Primary key</a> columns are the only columns implicitly included in secondary indexes. The inclusion of primary key columns improves performance when retrieving columns not in the index.</td></tr><tr><td><code>visible</code></td><td>Whether the index is visible to the <a href="/docs/v26.1/cost-based-optimizer#control-whether-the-optimizer-uses-an-index">cost-based optimizer</a>.</td></tr></tbody></table>

A column is in the primary key if the value of the `index_name` column is `{tbl}_pkey` and value of the `storing` column is `false`.

## Example

#### Setup

To follow along, run [`cockroach demo`](/docs/v26.1/cockroach-demo) to start a temporary, in-memory cluster with the [`movr`](/docs/v26.1/movr) sample dataset preloaded:

```shell
$ cockroach demo
```

### Show indexes for a table

```sql
> CREATE INDEX ON users (name);
```

```sql
> SHOW INDEX FROM users;
```

```text
  table_name |   index_name   | non_unique | seq_in_index | column_name | direction | storing | implicit | visible
-------------+----------------+------------+--------------+-------------+-----------+---------+----------+----------
  users      | users_name_idx |     t      |            1 | name        | ASC       |    f    |    f     |    t
  users      | users_name_idx |     t      |            2 | city        | ASC       |    f    |    t     |    t
  users      | users_name_idx |     t      |            3 | id          | ASC       |    f    |    t     |    t
  users      | users_pkey     |     f      |            1 | city        | ASC       |    f    |    f     |    t
  users      | users_pkey     |     f      |            2 | id          | ASC       |    f    |    f     |    t
  users      | users_pkey     |     f      |            3 | name        | N/A       |    t    |    f     |    t
  users      | users_pkey     |     f      |            4 | address     | N/A       |    t    |    f     |    t
  users      | users_pkey     |     f      |            5 | credit_card | N/A       |    t    |    f     |    t
(8 rows)
```

In this example, the columns where the value of the `index_name` column is `users_pkey` and value of the `storing` column is `false`, and thus are in the primary key, are `city` and `id`.
