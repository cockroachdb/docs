The `SHOW STATEMENT HINTS` [statement](/docs/v26.1/sql-statements) lists the [injected hints](/docs/v26.1/cost-based-optimizer#hint-injection) that have been created for a specific SQL statement fingerprint using the `information_schema.crdb_rewrite_inline_hints()` built-in function.

## Required privileges

Users must have the [`VIEWCLUSTERMETADATA`](/docs/v26.1/security-reference/authorization#supported-privileges) privilege to run `SHOW STATEMENT HINTS`.

## Response

The following fields are returned:

<table><thead><tr><th>Column</th><th>Type</th><th>Description</th></tr></thead><tbody><tr><td><code>row&#95;id</code></td><td><code>INT</code></td><td>A unique ID.</td></tr><tr><td><code>fingerprint</code></td><td><code>STRING</code></td><td>The SQL statement fingerprint that the hint applies to.</td></tr><tr><td><code>hint&#95;type</code></td><td><code>STRING</code></td><td>Hint type. <code>rewrite&#95;inline&#95;hints</code> indicates an injected hint.</td></tr><tr><td><code>created&#95;at</code></td><td><code>TIMESTAMPTZ</code></td><td>The timestamp when the injected hint was created.</td></tr><tr><td><code>details</code></td><td><code>JSONB</code></td><td>When the <a href="#options"><code>DETAILS</code></a> option is specified, hint-specific information in JSON format. For <code>rewrite&#95;inline&#95;hints</code>, this includes the donor SQL fingerprint with hints that will be applied.</td></tr></tbody></table>

## Examples

### Show hints for a statement

To show all injected hints for a specific statement fingerprint:

```sql
SHOW STATEMENT HINTS FOR $$ SELECT * FROM users WHERE city = _ $$;
```

```text
        row_id        |            fingerprint             |      hint_type       |          created_at
----------------------+------------------------------------+----------------------+--------------------------------
  1143470380756697089 | SELECT * FROM users WHERE city = _ | rewrite_inline_hints | 2026-01-21 21:11:06.782818+00
(1 row)
```

### Show hints with detailed information

To include the donor fingerprint in the output:

```sql
SHOW STATEMENT HINTS FOR $$ SELECT * FROM users WHERE city = _ $$ WITH DETAILS;
```

```text
        row_id        |            fingerprint             |      hint_type       |          created_at           |                              details
----------------------+------------------------------------+----------------------+-------------------------------+--------------------------------------------------------------------
  1143470380756697089 | SELECT * FROM users WHERE city = _ | rewrite_inline_hints | 2026-01-21 21:11:06.782818+00 | {"donorSql": "SELECT * FROM users@users_city_idx WHERE city = _"}
(1 row)
```
