The `RESET` [statement](/docs/v26.1/sql-statements) resets a [session variable](/docs/v26.1/set-vars) to its default value for the client session.

## Required privileges

No [privileges](/docs/v26.1/security-reference/authorization#managing-privileges) are required to reset a session setting.

## Example

<Tip>
You can use [`SET.. TO DEFAULT`](/docs/v26.1/set-vars#reset-a-variable-to-its-default-value) to reset a session variable as well.
</Tip>

```sql
> SET extra_float_digits = -10;
```

```sql
> SHOW extra_float_digits;
```

```text
 extra_float_digits
--------------------
 -10
(1 row)
```

```sql
> SELECT random();
```

```text
 random
---------
 0.20286
(1 row)
```

```sql
> RESET extra_float_digits;
```

```sql
> SHOW extra_float_digits;
```

```text
 extra_float_digits
--------------------
 0
(1 row)
```

```sql
> SELECT random();
```

```text
      random
-------------------
 0.561354028296755
(1 row)
```

## Known Limitations

- [`SET`](/docs/v26.1/set-vars) and `RESET` do not properly apply [`ROLLBACK`](/docs/v26.1/rollback-transaction) within a transaction. For example:

    ```sql
    SET TIME ZONE +2;
    BEGIN;
    SET TIME ZONE +3;
    ROLLBACK;
    SHOW TIME ZONE;
    ```

    ```sql
      timezone
    ------------
      <+03>-03
    ```

    ```sql
    SET TIME ZONE +2;
    BEGIN;
    RESET TIME ZONE;
    ROLLBACK;
    SHOW TIME ZONE;
    ```

    ```sql
      timezone
    ------------
      UTC
    ```

    [#69396](https://github.com/cockroachdb/cockroach/issues/69396), [#148766](https://github.com/cockroachdb/cockroach/issues/148766)
