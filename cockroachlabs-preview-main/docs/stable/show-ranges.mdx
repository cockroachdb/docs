The `SHOW RANGES` [statement](/docs/v26.1/sql-statements) shows information about the [ranges](/docs/v26.1/architecture/overview#architecture-range) that comprise the data for a table, index, database, or the current catalog. This information is useful for verifying how SQL data maps to underlying [ranges](/docs/v26.1/architecture/overview#architecture-range), and where the [replicas](/docs/v26.1/architecture/glossary#replica) for those ranges are located.

<Tip>
To show range information for a specific row in a table or index, use the [`SHOW RANGE... FOR ROW`](/docs/v26.1/show-range-for-row) statement.
</Tip>

## Required privileges

To use the `SHOW RANGES` statement, a user must either be a member of the [`admin`](/docs/v26.1/security-reference/authorization#admin-role) role (the `root` user belongs to the `admin` role by default) or have the `ZONECONFIG` [privilege](/docs/v26.1/security-reference/authorization#managing-privileges) defined.

## Options

The following [options](/docs/v26.1/sql-grammar#show_ranges_options) are available to affect the output. Multiple options can be passed at once, separated by commas.

- `TABLES`: List [tables](/docs/v26.1/show-tables) contained per [range](/docs/v26.1/architecture/overview#architecture-range).
- `INDEXES`: List [indexes](/docs/v26.1/indexes) contained per [range](/docs/v26.1/architecture/overview#architecture-range).
- `DETAILS`: Add [range](/docs/v26.1/architecture/overview#architecture-range) size, [leaseholder](/docs/v26.1/architecture/glossary#leaseholder) and other details. Note that this incurs a large computational overhead because it needs to fetch data across nodes.
- `KEYS`: Include binary [start and end keys](#start-key).

### Span Statistics

CockroachDB stores all user data in a sorted map of key-value pairs, also known as a keyspace. A span refers to an interval within this keyspace.

The `SHOW RANGES` command emits span statistics when the `DETAILS` option is specified. The statistics are included in a column named `span_stats`, as a `JSON` object.

The statistics are calculated for the identifier of each row. For example:

- `SHOW RANGES WITH DETAILS` will compute span statistics for each [range](/docs/v26.1/ui-replication-dashboard#review-of-cockroachdb-terminology).
- `SHOW RANGES WITH TABLES, DETAILS` will compute span statistics for each table.

The `span_stats` `JSON` object has the following keys:

<table><thead><tr><th>Key</th><th>Description</th></tr></thead><tbody><tr><td><code>approximate&#95;disk&#95;bytes</code></td><td>An approximation of the total on-disk size of the given object, across all <a href="/docs/v26.1/architecture/glossary#replica">replicas</a> (after compression).</td></tr><tr><td><code>key&#95;count</code></td><td>The number of non-system keys, including live and deleted keys, as well as new, uncommitted keys.</td></tr><tr><td><code>key&#95;bytes</code></td><td>The total key size (in bytes) of the keys tracked in <code>key&#95;count</code>.</td></tr><tr><td><code>val&#95;count</code></td><td>The number of values, or versions, of the keys tracked in <code>key&#95;count</code>, including deletion tombstones. A key will always have at least one value, but may have several historical values.</td></tr><tr><td><code>val&#95;bytes</code></td><td>Total value size (in bytes) of the values tracked in <code>val&#95;count</code>.</td></tr><tr><td><code>sys&#95;count</code></td><td>The number of system keys, which are not included in <code>key&#95;count</code>.</td></tr><tr><td><code>sys&#95;bytes</code></td><td>The total size (in bytes) of the keys in <code>sys&#95;count</code> and all of their values, or versions.</td></tr><tr><td><code>live&#95;count</code></td><td>The number of live (committed and non-deleted) non-system keys.</td></tr><tr><td><code>live&#95;bytes</code></td><td>Total size (in bytes) of the keys tracked in <code>live&#95;count</code> and their most recent value.</td></tr><tr><td><code>intent&#95;count</code></td><td>The number of <a href="/docs/v26.1/architecture/transaction-layer#write-intents">intents</a> (keys with new, uncommitted values).</td></tr><tr><td><code>intent&#95;bytes</code></td><td>The total size (in bytes) of the keys tracked in <code>intent&#95;count</code> and their uncommitted values.</td></tr></tbody></table>

## Examples

#### Setup

The following examples use MovR, a fictional vehicle-sharing application, to demonstrate CockroachDB SQL statements. For more information about the MovR example application and dataset, see [MovR: A Global Vehicle-sharing App](/docs/v26.1/movr).

To follow along, run [`cockroach demo`](/docs/v26.1/cockroach-demo) with the `--geo-partitioned-replicas` flag. This command opens an interactive SQL shell to a temporary, 9-node in-memory cluster with the `movr` database.

```shell
$ cockroach demo --geo-partitioned-replicas
```
