For PostgreSQL compatibility, CockroachDB includes a [system catalog](/docs/v26.1/system-catalogs) called `pg_catalog`. The tables in the `pg_catalog` schema roughly correspond to the [system catalogs in PostgreSQL](https://www.postgresql.org/docs/13/catalogs). `pg_catalog` tables are read-only.

## Data exposed by `pg_catalog`

The tables in CockroachDB's `pg_catalog` schema correspond to a subset of the virtual tables and views that make up the PostgreSQL system catalogs. Not all PostgreSQL system catalogs have a corresponding table in `pg_catalog`, and some of the `pg_catalog` tables are empty. To list the tables in `pg_catalog` for the [current database](/docs/v26.1/sql-name-resolution#current-database), use the following [`SHOW TABLES`](/docs/v26.1/show-tables) statement:

```sql
> SHOW TABLES FROM pg_catalog;
```

```text
  schema_name |       table_name        | type  | owner | estimated_row_count
--------------+-------------------------+-------+-------+----------------------
  pg_catalog  | pg_aggregate            | table | NULL  |                NULL
  pg_catalog  | pg_am                   | table | NULL  |                NULL
  ...
```

<Note>
To prohibit queries against empty tables, set the `stub_catalog_tables` [session variable](/docs/v26.1/set-vars) to `off`.
</Note>

## Query `pg_catalog` tables

You can run [`SELECT` queries](/docs/v26.1/selection-queries) on the tables in `pg_catalog`.

<Tip>
To ensure that you can view all of the tables in `pg_catalog`, query the tables as a user with [`admin` privileges](/docs/v26.1/security-reference/authorization#admin-role).
</Tip>

<Note>
Unless specified otherwise, queries to `pg_catalog` assume the [current database](/docs/v26.1/sql-name-resolution#current-database).
</Note>

For example, to return the `pg_catalog` table with additional information about indexes in [`movr` database](/docs/v26.1/movr), you can query the `pg_catalog.pg_indexes` table:

```sql
> SELECT * FROM movr.pg_catalog.pg_indexes;
```
