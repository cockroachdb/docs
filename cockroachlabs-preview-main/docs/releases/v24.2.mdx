### Changelog

View a detailed changelog on GitHub: [v24.2.9...v24.2.10](https://github.com/cockroachdb/cockroach/compare/v24.2.9...v24.2.10)

### General changes

- The protected timestamp (PTS) records of running changefeeds are now updated when the set of targets changes, such as when system tables are added to the protected tables list. [#138653](https://github.com/cockroachdb/cockroach/pull/138653)

### SQL language changes

- The `legacy_varchar_typing` session setting has been added, which reverts the changes of [#133037](https://github.com/cockroachdb/cockroach/pull/133037) that cause the change in typing behavior described in [#137837](https://github.com/cockroachdb/cockroach/pull/137837). Specifically, it makes type-checking and overload resolution ignore the newly added "unpreferred" overloads. This setting defaults to `on`. [#138002](https://github.com/cockroachdb/cockroach/pull/138002)
- Since v23.2 table statistics histograms had been collected for non-indexed JSON columns. Histograms are no longer collected for these columns if the `sql.stats.non_indexed_json_histograms.enabled` cluster setting is set to `false`. This reduces memory usage during table statistics collection, for both automatic and manual collection via `ANALYZE` and `CREATE STATISTICS`. [#140267](https://github.com/cockroachdb/cockroach/pull/140267)
- Added support for a new index hint, `AVOID_FULL_SCAN`, which will prevent the optimizer from planning a full scan for the specified table if any other plan is possible. The hint can be used in the same way as other existing index hints. For example, `SELECT * FROM table_name@{AVOID_FULL_SCAN};`. This hint is similar to `NO_FULL_SCAN`, but will not error if a full scan cannot be avoided. Note that normally a full scan of a partial index would not be considered a "full scan" for the purposes of the `NO_FULL_SCAN` and `AVOID_FULL_SCAN` hints, but if the user has explicitly forced the partial index via `FORCE_INDEX=index_name`, CockroachDB does consider it a full scan. [#140271](https://github.com/cockroachdb/cockroach/pull/140271)
- Added the `optimizer_prefer_bounded_cardinality` session setting, which instructs the optimizer to prefer query plans where every expression has a guaranteed upper-bound on the number of rows it will process. This may help the optimizer produce better query plans in some cases. This setting is disabled by default. [#140271](https://github.com/cockroachdb/cockroach/pull/140271)
- Added the `optimizer_min_row_count` session setting, which sets a lower bound on row count estimates for relational expressions during query planning. A value of `0`, which is the default, indicates no lower bound. Note that if this is set to a value greater than `0`, a row count of zero can still be estimated for expressions with a cardinality of zero, e.g., for a contradictory filter. Setting this to a value higher than `0`, such as `1`, may yield better query plans in some cases, such as when statistics are frequently stale and inaccurate. [#140271](https://github.com/cockroachdb/cockroach/pull/140271)

### Operational changes

- Schema object identifiers (e.g., database names, schema names, table names, function names, and type names) are no longer redacted when logging statements in the `EXEC` or `SQL_SCHEMA` log channels. If redaction of these names is required, then the new cluster setting `sql.log.redact_names.enabled` can be set to `true`. The default value of the setting is `false`. [#138656](https://github.com/cockroachdb/cockroach/pull/138656)
- Added a metric, `sql.schema_changer.object_count`, that keeps track of the count of schema objects in the cluster. [#138838](https://github.com/cockroachdb/cockroach/pull/138838)
- The `changefeed.max_behind_nanos` metric now supports scoping with metrics labels. [#139233](https://github.com/cockroachdb/cockroach/pull/139233)

### Bug fixes

- `CLOSE CURSOR` statements are now allowed in read-only transactions, similar to PostgreSQL. This bug had been present since at least v23.1. [#137791](https://github.com/cockroachdb/cockroach/pull/137791)
- `ALTER BACKUP SCHEDULE` no longer fails on schedules whose collection URI contains a space. [#138081](https://github.com/cockroachdb/cockroach/pull/138081)
- Previously, `SHOW CREATE TABLE` was showing incorrect data with regard to inverted indexes. It now shows the correct data in a format that can be repeatedly entered back into CockroachDB to recreate the same table. [#138084](https://github.com/cockroachdb/cockroach/pull/138084)
- Fixed a timing issue between `ALTER VIEW.. RENAME` and `DROP VIEW` that caused repeated failures in the `DROP VIEW` job. [#137888](https://github.com/cockroachdb/cockroach/pull/137888)
- Fixed a bug where querying the `pg_catalog.pg_constraint` table while the schema changer was dropping a constraint could result in a query error. [#138284](https://github.com/cockroachdb/cockroach/pull/138284)
- Queries that perform a cast from the string representation of an array containing `GEOMETRY` or `GEOGRAPHY` types to a SQL `ARRAY` type will now succeed. [#138694](https://github.com/cockroachdb/cockroach/pull/138694)
- Fixed a bug where secondary tenants could fatal when issuing HTTP requests during tenant startup. [#138754](https://github.com/cockroachdb/cockroach/pull/138754)
- Fixed a bug where CockroachDB could encounter an internal error `comparison of two different versions of enum` in some cases when a user-defined type was modified within a transaction and subsequent statements read the column of that user-defined type. The bug was introduced in v24.2. [#138053](https://github.com/cockroachdb/cockroach/pull/138053)
- When the session variable `allow_role_memberships_to_change_during_transaction` is set, it is now possible to create and drop users quickly even when there are contending transactions on the `system.users` and `system.role_options` system tables. [#139031](https://github.com/cockroachdb/cockroach/pull/139031)
- Fixed a bug where the error `batch timestamp... must be after replica GC threshold` could occur during a schema change backfill operation, and cause the schema change job to retry infinitely. Now this error is treated as permanent, and will cause the job to enter the `failed` state. [#139249](https://github.com/cockroachdb/cockroach/pull/139249)

### Changelog

View a detailed changelog on GitHub: [v24.2.7...v24.2.8](https://github.com/cockroachdb/cockroach/compare/v24.2.7...v24.2.8)

### General changes

- To improve the granularity of changefeed pipeline metrics, the changefeed metrics `changefeed.admit_latency` and `changefeed.commit_latency` have histogram buckets from `5ms` to `60m` (previously `500ms` to `5m` ). The following changefeed metrics have histogram buckets from `5ms` to `10m` (previously `500ms` to `5m` ):
    - `changefeed.parallel_io_queue_nanos`
    - `changefeed.parallel_io_result_queue_nanos`
    - `changefeed.sink_batch_hist_nanos`
    - `changefeed.flush_hist_nanos`
    - `changefeed.kafka_throttling_hist_nanos` [#136603](https://github.com/cockroachdb/cockroach/pull/136603)
- Added support for multiple seed brokers in the new Kafka sink. [#136748](https://github.com/cockroachdb/cockroach/pull/136748)
- Added a new metric ( `distsender.rangefeed.catchup_ranges_waiting_client_side` ) that counts how many rangefeeds are waiting on the client-side limiter to start performing catchup scans. [#136835](https://github.com/cockroachdb/cockroach/pull/136835)
- Added changefeed support for the `mvcc_timestamp` option with the `avro` format. If both options are specified, the Avro schema includes an `mvcc_timestamp` metadata field and emits the row's MVCC timestamp with the row data. [#136016](https://github.com/cockroachdb/cockroach/pull/136016)
- Added a no-op `AWS_USE_PATH_STYLE` parameter for forward compatibility with v24.3. [#137025](https://github.com/cockroachdb/cockroach/pull/137025)

### SQL language changes

- Added the `legacy_varchar_typing` session setting, which reverts the changes of [#133037](https://github.com/cockroachdb/cockroach/pull/133037) that causes the change in typing behavior described in [#137837](https://github.com/cockroachdb/cockroach/pull/137837). Specifically, it makes type-checking and overload resolution ignore the newly added "unpreferred" overloads. This setting defaults to `on`. [#137920](https://github.com/cockroachdb/cockroach/pull/137920)

### Operational changes

- Removed the `sql.auth.resolve_membership_single_scan.enabled` cluster setting. This was added out of precaution in case it was necessary to revert back to the old behavior for looking up role memberships, but this escape hatch has never been needed in practice since this was added in v23.1. [#136161](https://github.com/cockroachdb/cockroach/pull/136161)
- Telemetry delivery is now considered successful even in cases where we experience a network timeout. This will prevent throttling in cases outside an operator's control. [#136479](https://github.com/cockroachdb/cockroach/pull/136479)
- When a schema change job is completed, rolls back, or encounters a failure, the time taken since the job began is now logged in a structured log in the `SQL_SCHEMA` log channel. [#136928](https://github.com/cockroachdb/cockroach/pull/136928)
- Added a new configurable parameter `kv.transaction.max_intents_and_locks` that will prevent transactions from creating too many intents. [#137700](https://github.com/cockroachdb/cockroach/pull/137700)
- Added the metric `txn.count_limit_rejected`, which tracks the KV transactions that have been aborted because they exceeded the max number of writes and locking reads allowed. [#137700](https://github.com/cockroachdb/cockroach/pull/137700)
- Added the metric `txn.count_limit_on_response`, which tracks the number of KV transactions that have exceeded the count limit on a response. [#137700](https://github.com/cockroachdb/cockroach/pull/137700)

### Bug fixes

- `ALTER COLUMN SET NOT NULL` was not enforced consistently when the table was created in the same transaction. [#136365](https://github.com/cockroachdb/cockroach/pull/136365)
- Fixed a bug where `CREATE RELATION / TYPE` could leave dangling namespace entries if the schema was concurrently being dropped. [#136379](https://github.com/cockroachdb/cockroach/pull/136379)
- The `idle_in_session_timeout` setting now excludes the time spent waiting for schema changer jobs to complete, preventing unintended session termination during schema change operations. [#136502](https://github.com/cockroachdb/cockroach/pull/136502)
- Fixed a bug that causes the optimizer to use stale table statistics after altering an `ENUM` type used in the table. [#136759](https://github.com/cockroachdb/cockroach/pull/136759)
- Table statistics collection in CockroachDB could previously run into `no bytes in account to release` errors in some edge cases (when the SQL memory budget, configured via `--max-sql-memory` flag, was close to being exhausted). The bug has been present since v21.2 and is now fixed. [#136165](https://github.com/cockroachdb/cockroach/pull/136165)
- CockroachDB now better respects the `statement_timeout` limit on queries involving the top K sort and merge join operations. [#136652](https://github.com/cockroachdb/cockroach/pull/136652)
- Fixed an issue where license enforcement was not consistently disabled for single-node clusters started with `cockroach start-single-node`. The fix ensures proper behavior on cluster restarts. [#137011](https://github.com/cockroachdb/cockroach/pull/137011)
- Fixed a bug that caused queries against tables with user-defined types to sometimes fail with errors after restoring those tables. [#137354](https://github.com/cockroachdb/cockroach/pull/137354)
- Fixed a bug that caused an incorrect filesystem to be logged as part of the store information. [#137113](https://github.com/cockroachdb/cockroach/pull/137113)
- Fixed a bug that has existed since v24.1 that would cause a set-returning UDF with `OUT` parameters to return a single row. [#137377](https://github.com/cockroachdb/cockroach/pull/137377)
- Fixed a bug that could cause an internal error if a table with an implicit ( `rowid` ) primary key was locked from within a subquery, like: `SELECT * FROM (SELECT * FROM foo WHERE x = 2) FOR UPDATE;`. The error could occur either under read-committed isolation, or with `optimizer_use_lock_op_for_serializable` enabled. [#137129](https://github.com/cockroachdb/cockroach/pull/137129)
- Fixed an issue where adding an existing column with the `IF NOT EXISTS` option could exit too early, skipping necessary handling of the abstract syntax tree (AST). This could lead to failure of the `ALTER` statement. [#137676](https://github.com/cockroachdb/cockroach/pull/137676)
- Fixed a bug related to displaying the names of composite types in the `SHOW CREATE TABLES` command. The names are now shown as two-part names, which disambiguates the output and makes it more portable to other databases. [#137768](https://github.com/cockroachdb/cockroach/pull/137768)
- Fixed an issue where a schema change could incorrectly cause a changefeed to fail with an assertion error like `received boundary timestamp... of type... before reaching existing boundary of type...`. [#137705](https://github.com/cockroachdb/cockroach/pull/137705)
- Internal scans are now exempt from the `sql.defaults.disallow_full_table_scans.enabled` setting, allowing index creation even when the cluster setting is active. [#137724](https://github.com/cockroachdb/cockroach/pull/137724)

### Performance improvements

- Improved the internal caching logic for role membership information. This reduces the latency impact of commands such as `DROP ROLE`, `CREATE ROLE`, and `GRANT role TO user`, which cause the role membership cache to be invalidated. [#136161](https://github.com/cockroachdb/cockroach/pull/136161)

### Security updates

- All cluster settings that accept strings are now fully redacted when transmitted as part of diagnostics telemetry. This payload includes a record of modified cluster settings and their values when they are not strings. Customers who previously applied the mitigations in [Technical Advisory 133479](/docs/advisories/a133479) can safely set the value of cluster setting `server.redact_sensitive_settings.enabled` to false and turn on diagnostic reporting via the `diagnostics.reporting.enabled` cluster setting without leaking sensitive cluster settings values. [#134017](https://github.com/cockroachdb/cockroach/pull/134017)

### General changes

- `COCKROACH_SKIP_ENABLING_DIAGNOSTIC_REPORTING` is no longer mentioned in the `cockroach demo` command. [#134088](https://github.com/cockroachdb/cockroach/pull/134088)
- Added `system.users` to the list of system tables that changefeeds protect with protected timestamps. This table is required for change data capture queries. [#134837](https://github.com/cockroachdb/cockroach/pull/134837)

### Operational changes

- The `goschedstats.always_use_short_sample_period.enabled` setting should be set to true for any production cluster, to prevent unnecessary queuing in admission control CPU queues. [#133584](https://github.com/cockroachdb/cockroach/pull/133584)
- Added a new cluster setting `ui.database_locality_metadata.enabled` that allows operators to disable loading extended database and table region information in the DB Console Database and Table pages. This information can cause significant CPU load on large clusters with many ranges. Versions of this page from v24.3 and later do not have this problem. If customers require this data, they can use the `SHOW RANGES FROM {DATABASE| TABLE}` query via SQL to compute on-demand. [#134095](https://github.com/cockroachdb/cockroach/pull/134095)
- Row-level TTL jobs now periodically log progress by showing the number of table spans that have been processed so far. [#135170](https://github.com/cockroachdb/cockroach/pull/135170)

### Bug fixes

- Fixed a bug that caused non-reusable query plans, e.g., plans for DDL and `SHOW...` statements, to be cached and reused in future executions, possibly causing stale results to be returned. This bug only occurred when `plan_cache_mode` was set to `auto` or `force_generic_plan`, both of which are not currently the default settings. [#133074](https://github.com/cockroachdb/cockroach/pull/133074)
- Previously, CockroachDB could encounter an internal error of the form `interface conversion: coldata.Column is` in an edge case and this is now fixed. The bug is present in v22.2.13+, v23.1.9+, v23.2+. [#133761](https://github.com/cockroachdb/cockroach/pull/133761)
- Fixed a bug that caused incorrect `NOT NULL` constraint violation errors on `UPSERT` and `INSERT.. ON CONFLICT.. DO UPDATE` statements when those statements updated an existing row and a subset of columns that did not include a `NOT NULL` column of the table. This bug has been present since at least v20.1.0. [#133821](https://github.com/cockroachdb/cockroach/pull/133821)
- Fixed an unhandled error that could occur when using `REVOKE... ON SEQUENCE FROM... user` on an object that is not a sequence. [#133709](https://github.com/cockroachdb/cockroach/pull/133709)
- Addressed a panic inside `CREATE TABLE AS` if sequence builtin expressions had invalid function overloads. [#133869](https://github.com/cockroachdb/cockroach/pull/133869)
- `STRING` constants can now be compared against collated strings. [#134084](https://github.com/cockroachdb/cockroach/pull/134084)
- When executing queries with index / lookup joins when the ordering needs to be maintained, previously CockroachDB could experience increased query latency, possibly by several orders of magnitude. This bug was introduced in v22.2 and is now fixed. [#134366](https://github.com/cockroachdb/cockroach/pull/134366)
- Fixed a minor bug where `DISCARD ALL` statements were counted under the `sql.ddl.count` metric. Now these will be counted under the `sql.misc.count` metric. [#134509](https://github.com/cockroachdb/cockroach/pull/134509)
- Addressed a bug with `DROP CASCADE` that would occasionally panic with an undropped `backref` message on partitioned tables. [#134472](https://github.com/cockroachdb/cockroach/pull/134472)
- Reduced the duration of partitions in the gossip network when a node crashes in order to eliminate false positives in the `ranges.unavailable` metric. [#134600](https://github.com/cockroachdb/cockroach/pull/134600)
- Non- `admin` users that run `DROP ROLE IF EXISTS` on a user that does not exist will no longer receive an error message. [#134969](https://github.com/cockroachdb/cockroach/pull/134969)
- Fixed a bug that caused quotes around the name of a routine to be dropped when it was called within another routine. This could prevent the correct routine from being resolved if the nested routine name was case sensitive. The bug has existed since v24.1, when nested routines were introduced. [#134000](https://github.com/cockroachdb/cockroach/pull/134000)
- Fixed a bug that could cause incorrect query results when the optimizer planned a lookup join on an index containing a column of type `CHAR(N)`, `VARCHAR(N)`, `BIT(N)`, `VARBIT(N)`, or `DECIMAL(M, N)`, and the query held that column constant to a single value (e.g., with an equality filter). [#135076](https://github.com/cockroachdb/cockroach/pull/135076)
- Fixed an unhandled error that would occur if `DROP SCHEMA` was executed on the `public` schema of the `system` database, or on an internal schema like `pg_catalog` or `information_schema`. [#135180](https://github.com/cockroachdb/cockroach/pull/135180)
- Fixed a bug that caused incorrect evaluation of some binary expressions involving `CHAR(N)` values and untyped string literals with trailing whitespace characters. For example, the expression `'f'::CHAR = 'f '` now correctly evaluates to `true`. [#135689](https://github.com/cockroachdb/cockroach/pull/135689)

- `CREATE SCHEMA` now returns the correct error if a the schema name is missing. [#135927](https://github.com/cockroachdb/cockroach/pull/135927)
- Using more than one `DECLARE` statment in the definition of a user-defined function now correctly declares additional variables. [#135738](https://github.com/cockroachdb/cockroach/pull/135738)
- Fixed a bug where CockroachDB would encounter an internal error when evaluating `FETCH ABSOLUTE 0` statements. The bug has been present since v22.1. [#134992](https://github.com/cockroachdb/cockroach/pull/134992)
- Fixed an issue where corrupted table statistics could cause the `cockroach` process to crash. [#136041](https://github.com/cockroachdb/cockroach/pull/136041)

### Enterprise edition changes

A changefeed now only updates its PTS if `changefeed.protect_timestamp.lag` has passed between the last PTS and the changefeed high-water mark.     - [SQL sink](https://www.cockroachlabs.com/docs/v24.2/changefeed-for). If [child metrics](https://www.cockroachlabs.com/docs/v24.2/child-metrics) are enabled, the metric will have a `sql` label. [#130582](https://github.com/cockroachdb/cockroach/pull/130582)

### Security updates

- The new cluster setting `server.jwt_authentication.issuers.configuration` is now aliased to [`server.jwt_authentication.issuers`](https://www.cockroachlabs.com/docs/v24.2/sso-sql#cluster-settings). The new cluster setting reflects the value the setting can take. The setting can now take multiple values to support various kinds of providers and their mapped JWKS URIs. This can be set to one of the following values:
    - Simple string that Go can parse as a valid issuer URL: `'https://accounts.google.com'`.
    - String that can be parsed as a valid JSON array of issuer URLs list: `['example.com/adfs','https://accounts.google.com']`.
    - String that can be parsed as a valid JSON and deserialized into a map of issuer URLs to corresponding JWKS URIs. In this case, CockroachDB will override the JWKS URI present in the issuer's well-known endpoint: `'{ "issuer_jwks_map": { "https://accounts.google.com": "https://www.googleapis.com/oauth2/v3/certs", "example.com/adfs": "https://example.com/adfs/discovery/keys" } }'`.

        When `issuer_jwks_map` is set, CockroachDB directly uses the JWKS URI to get the key set. In all other cases where [`server.jwt_authentication.jwks_auto_fetch.enabled`](https://www.cockroachlabs.com/docs/v24.2/sso-sql#cluster-settings) is set, CockroachDB attempts to automatically obtain the JWKS URI first from the issuer's well-known endpoint. [#128178](https://github.com/cockroachdb/cockroach/pull/128178)

### Enterprise edition changes

- The new Kafka [changefeed sink](https://www.cockroachlabs.com/docs/v24.2/changefeed-sinks) is now enabled by default. ### Operational changes

- The cluster setting [`storage.ingestion.value_blocks.enabled`](https://www.cockroachlabs.com/docs/v24.2/cluster-settings#setting-storage-ingestion-value-blocks-enabled) can be set to `false` if a pathological huge [range](https://www.cockroachlabs.com/docs/v24.2/architecture/glossary#range) happens to occur in a cluster, and incoming [snapshots](https://www.cockroachlabs.com/docs/v24.2/architecture/replication-layer#snapshots) of that range are causing [OOMs](https://www.cockroachlabs.com/docs/v24.2/cluster-setup-troubleshooting#out-of-memory-oom-crash). [#128098](https://github.com/cockroachdb/cockroach/pull/128098)

    - The [`node_shutdown_connection_timeout`](https://www.cockroachlabs.com/docs/v24.2/eventlog#node_shutdown_connection_timeout) event is logged after the timeout defined by [`server.shutdown.connections.timeout`](https://www.cockroachlabs.com/docs/v24.2/cluster-settings#setting-server-shutdown-connection-wait) transpires, if there are still [open SQL connections](https://www.cockroachlabs.com/docs/v24.2/show-sessions).
    - The [`node_shutdown_transaction_timeout`](https://www.cockroachlabs.com/docs/v24.2/eventlog#node_shutdown_transaction_timeout) event is logged after the timeout defined by [`server.shutdown.transactions.timeout`](https://www.cockroachlabs.com/docs/v24.2/cluster-settings#setting-server-shutdown-query-wait) transpires, if there are still open [transactions](https://www.cockroachlabs.com/docs/v24.2/transactions) on those SQL connections. ### Bug fixes

- Fixed a memory leak that could occur when a connection string specifies a [virtual cluster](https://www.cockroachlabs.com/docs/v24.2/cluster-virtualization-overview) that does not exist. [#128108](https://github.com/cockroachdb/cockroach/pull/128108)
- Fixed a bug where [`CREATE INDEX IF NOT EXISTS`](https://www.cockroachlabs.com/docs/v24.2/create-index) would not correctly short-circuit if the given index already existed. [#128240](https://github.com/cockroachdb/cockroach/pull/128240)

- Fixed a bug where an [index](https://www.cockroachlabs.com/docs/v24.2/indexes) could store a column in the primary index if that column had a mixed-case name. [#128240](https://github.com/cockroachdb/cockroach/pull/128240)
- Setting or dropping a default value on a [computed column](https://www.cockroachlabs.com/docs/v24.2/computed-columns) is now disallowed -- even for null defaults. Previously, setting or dropping a default value on a computed column was a no-op; now it is an error. [#128465](https://github.com/cockroachdb/cockroach/pull/128465)
- Fixed a bug where a hash-sharded constraint could not be created if it referred to a column that had a backslash in its name. [#128522](https://github.com/cockroachdb/cockroach/pull/128522)
- Fixed a bug introduced in v23.1 where the output of [`EXPLAIN (OPT, REDACT)`](https://www.cockroachlabs.com/docs/v24.2/explain) for various `CREATE` statements was not redacted. This bug affects the following statements:
    - `EXPLAIN (OPT, REDACT) CREATE TABLE`
    - `EXPLAIN (OPT, REDACT) CREATE VIEW`
    - `EXPLAIN (OPT, REDACT) CREATE FUNCTION` [#128490](https://github.com/cockroachdb/cockroach/pull/128490)
- Fixed a bug where legacy and [declarative schema changes](https://www.cockroachlabs.com/docs/v24.2/online-schema-changes) could be executed concurrently, which could lead to failing or hung schema change jobs. [#128825](https://github.com/cockroachdb/cockroach/pull/128825)
- Fixed a bug that caused errors like `ERROR: column 'crdb_internal_idx_expr' does not exist` when accessing a table with an [expression index](https://www.cockroachlabs.com/docs/v24.2/expression-indexes) where the expression evaluates to an [ENUM type](https://www.cockroachlabs.com/docs/v24.2/enum), such as `CREATE INDEX ON t ((col::an_enum))`. [#129094](https://github.com/cockroachdb/cockroach/pull/129094)

##### Settings added

- `debug.zip.redact_addresses`: Added the [cluster setting](https://www.cockroachlabs.com/docs/v24.2/cluster-settings) `debug.zip.redact_addresses.enabled` that allows the user to enable or disable redaction of fields like `hostname` and IP addresses. [#123544](https://github.com/cockroachdb/cockroach/pull/123544)
- `kv.transaction.randomized_anchor_key`: Previously, concurrent transactions that constructed large write batches could cause [hotspots](https://www.cockroachlabs.com/docs/v24.2/ui-hot-ranges-page). This was because the [transaction record](https://www.cockroachlabs.com/docs/v24.2/architecture/transaction-layer#transaction-records) for all [ranges](https://www.cockroachlabs.com/docs/v24.2/architecture/reads-and-writes-overview#range) would coalesce on a single range, which would then cause this range's [leaseholder](https://www.cockroachlabs.com/docs/v24.2/architecture/reads-and-writes-overview#leaseholder) to perform all intent resolution work. This is fixed by distributing transaction records randomly across the ranges the write batch touches. In turn, hotspots are prevented. [#125744](https://github.com/cockroachdb/cockroach/pull/125744)
- `server.oidc_authentication.client.timeout`: The new [cluster setting](https://www.cockroachlabs.com/docs/v24.2/cluster-settings#setting-server-oidc-authentication-client-timeout) `server.oidc_authentication.client.timeout` allows configuration of the HTTP client timeout for external requests made during [OIDC authentication](https://www.cockroachlabs.com/docs/v24.2/sso-db-console). The default timeout is 30 seconds. [#125767](https://github.com/cockroachdb/cockroach/pull/125767)
- `sql.auth.grant_option_for_owner.enabled`: The new [cluster setting](https://www.cockroachlabs.com/docs/v24.2/cluster-settings) [`sql.auth.grant_option_for_owner.enabled`](https://www.cockroachlabs.com/docs/v24.2/cluster-settings#setting-sql-auth-grant-option-for-owner-enabled) controls whether the owner of an object has permission to grant permission on the object to other [users](https://www.cockroachlabs.com/docs/v24.2/security-reference/authorization#roles). When set to `true` (the default), the [`GRANT OPTION`](https://www.cockroachlabs.com/docs/v24.2/show-grants#privilege-grants) is implicitly granted to the object owner, who can grant permissions on the object to other users, preserving the existing behavior of CockroachDB. When set to `false`, the `GRANT OPTION` is not implicitly given to the owner of an object. The owner's permissions do not change, but they can no longer grant permissions to others unless the `GRANT OPTION` is granted to them explicitly. [#126960](https://github.com/cockroachdb/cockroach/pull/126960)
- `sql.auth.grant_option_inheritance.enabled`: Added the [`sql.auth.grant_option_inheritance.enabled` cluster setting](https://www.cockroachlabs.com/docs/v24.2/cluster-settings#setting-sql-auth-grant-option-inheritance-enabled). The default value is `true`, which maintains consistency with CockroachDB's previous behavior: users granted a privilege with [`WITH GRANT OPTION`](https://www.cockroachlabs.com/docs/v24.2/grant) can in turn grant that privilege to others. When `sql.auth.grant_option_inheritance.enabled` is set to `false`, the `GRANT OPTION` is not inherited through role membership, thereby preventing descendant roles from granting the privilege to others. However, the privilege itself continues to be inherited through role membership. [#125590](https://github.com/cockroachdb/cockroach/pull/125590)
- `storage.sstable.compression_algorithm_backup_storage`, `storage.sstable.compression_algorithm_backup_transport`: Added two new [cluster settings](https://www.cockroachlabs.com/docs/v24.2/cluster-settings), `storage.sstable.compression_algorithm_backup_storage` and `storage.sstable.compression_algorithm_backup_transport`, which in addition to the existing cluster setting `storage.sstable.compression_algorithm`, can be used to alter the compression algorithm used for various types of [SSTs](https://www.cockroachlabs.com/docs/v24.2/architecture/storage-layer#ssts). [#124245](https://github.com/cockroachdb/cockroach/pull/124245)

##### Settings removed

- `kv.rangefeed.range_stuck_threshold`: Removed the stuck rangefeed cancel feature and its related [cluster setting](https://www.cockroachlabs.com/docs/v24.2/cluster-settings) `kv.rangefeed.range_stuck_threshold`, because it was only available in [non-mux rangefeeds](https://www.cockroachlabs.com/docs/v24.2/advanced-changefeed-configuration#mux-rangefeeds). Previously, the stuck rangefeed cancel feature was introduced to restart single rangefeeds automatically if they had not received KV updates for some time. [#125663](https://github.com/cockroachdb/cockroach/pull/125663)
- `storage.value_blocks.enabled`: The `storage.value_blocks.enabled` [cluster setting](https://www.cockroachlabs.com/docs/v24.2/cluster-settings) has been removed; value blocks are always enabled. [#122164](https://github.com/cockroachdb/cockroach/pull/122164)

##### Settings with changed defaults

- `kv.dist_sender.circuit_breakers.mode` has had its default changed to `liveness range only = 1`.
- `sql.defaults.results_buffer.size` has had its default changed to `512 KiB`: The default value of the `sql.defaults.results_buffer.size` [cluster setting](https://www.cockroachlabs.com/docs/v24.2/cluster-settings) has been changed from 16KiB to 512KiB. This reduces the chance that clients using [`READ COMMITTED`](https://www.cockroachlabs.com/docs/v24.2/read-committed) transactions will encounter errors that cannot automatically be retried within CockroachDB. [#124633](https://github.com/cockroachdb/cockroach/pull/124633)
- `sql.metrics.max_mem_stmt_fingerprints` and `sql.metrics.max_mem_txn_fingerprints` have had their defaults changed to `7500`: The default values for the [cluster settings](https://www.cockroachlabs.com/docs/v24.2/cluster-settings) `sql.metrics.max_mem_stmt_fingerprints` and `sql.metrics.max_mem_txn_fingerprints` have been changed from `100000` to `7500`, thus lowering the default limits for in-memory statement and transaction fingerprints. [#123430](https://github.com/cockroachdb/cockroach/pull/123430)
- `sql.stats.histogram_samples.count` has had its default changed to `0`: Histograms are no longer constructed using a default sample size of `10k`. Samples are now sized dynamically based on table size unless the sample count has been set in the table or [cluster settings](https://www.cockroachlabs.com/docs/v24.2/cluster-settings). [#125345](https://github.com/cockroachdb/cockroach/pull/125345)
- `sql.ttl.default_delete_rate_limit` has had its default changed to `100`: The storage parameter `ttl_delete_rate_limit`, which determines the rate limit for deleting expired rows, is now set to `100` by default. [#124354](https://github.com/cockroachdb/cockroach/pull/124354)

##### Settings with new options

- `storage.sstable.compression_algorithm` has added the option `none = 3`: The compression option `none` was added to allow for the disabling of SSTable compression. This option is disabled by default, but can can be used with any of the three existing cluster settings that control SSTable compression: `storage.sstable.compression_algorithm`, `storage.sstable.compression_algorithm_backup_storage`, and `storage.sstable.compression_algorithm_backup_transport`. [#126508](https://github.com/cockroachdb/cockroach/pull/126508)

[#127760](https://github.com/cockroachdb/cockroach/pull/127760)
- Fixed a bug in [debug zip](https://www.cockroachlabs.com/docs/v24.2/cockroach-debug-zip) generation where an error was produced while fetching unstructured/malformed [logs](https://www.cockroachlabs.com/docs/v24.2/log-formats). [#127883](https://github.com/cockroachdb/cockroach/pull/127883)
- Fixed small memory leaks that occur during [changefeed creation](https://www.cockroachlabs.com/docs/v24.2/create-changefeed). [#127899](https://github.com/cockroachdb/cockroach/pull/127899)
- Fixed a [known limitation](https://www.cockroachlabs.com/docs/v24.2/physical-cluster-replication-overview#known-limitations) in which [fast cutback](https://www.cockroachlabs.com/docs/v24.2/cutover-replication#cut-back-to-the-original-primary-cluster) could fail. Users can now protect data for the [default protection window](https://www.cockroachlabs.com/docs/v24.2/physical-cluster-replication-technical-overview) of 4 hours on both the primary and the standby clusters. [#127892](https://github.com/cockroachdb/cockroach/pull/127892)

    - [`storage.sstable.compression_algorithm_backup_transport`](https://www.cockroachlabs.com/docs/v24.2/cluster-settings#setting-storage-sstable-compression-algorithm-backup-transport) [#126508](https://github.com/cockroachdb/cockroach/pull/126508)

### SQL language changes

- Added the [`sql.auth.grant_option_inheritance.enabled` cluster setting](https://www.cockroachlabs.com/docs/v24.2/cluster-settings#setting-sql-auth-grant-option-inheritance-enabled). The default value is `true`, which maintains consistency with CockroachDB's previous behavior: users granted a privilege with [`WITH GRANT OPTION`](https://www.cockroachlabs.com/docs/v24.2/grant) can in turn grant that privilege to others. When `sql.auth.grant_option_inheritance.enabled` is set to `false`, the `GRANT OPTION` is not inherited through role membership, thereby preventing descendant roles from granting the privilege to others. However, the privilege itself continues to be inherited through role membership. [#125590](https://github.com/cockroachdb/cockroach/pull/125590)
- The [`pg_catalog.pg_attribute`](https://www.cockroachlabs.com/docs/v24.2/pg-catalog) table now has a column named `attishidden`, which indicates if the table column or attribute is [`NOT VISIBLE`](https://www.cockroachlabs.com/docs/v24.2/create-table#not-visible-property). [#126397](https://github.com/cockroachdb/cockroach/pull/126397)

This change makes it easier to apply the output with enum types to other databases. [#125996](https://github.com/cockroachdb/cockroach/pull/125996)
- When [altering the data type of columns](https://www.cockroachlabs.com/docs/v24.2/alter-table#alter-column) with the [hidden attribute (`NOT VISIBLE`)](https://www.cockroachlabs.com/docs/v24.2/create-table#not-visible-property), the alteration now preserves the hidden attribute in the column. [#126033](https://github.com/cockroachdb/cockroach/pull/126033)

[#126281](https://github.com/cockroachdb/cockroach/pull/126281)

The bug has existed since v22.2 when [user-defined functions (UDFs)](https://www.cockroachlabs.com/docs/v24.2/user-defined-functions) were introduced. - Fixed a bug when inputting `public` role as user name for [built-in compatibility functions](https://www.cockroachlabs.com/docs/v24.2/functions-and-operators#compatibility-functions), such as `has_database_privilege` and `has_schema_privilege`. ### Performance improvements

- More efficient [query plans](https://www.cockroachlabs.com/docs/v24.2/cost-based-optimizer) are now generated for queries with [text similarity filters](https://www.cockroachlabs.com/docs/v24.2/trigram-indexes#how-do-trigram-indexes-work), e.g., `text_col % 'foobar'`. These plans are generated if the `optimizer_use_trigram_similarity_optimization` [session setting](https://www.cockroachlabs.com/docs/v24.2/session-variables) is enabled, which it is by default. This setting is disabled by default in previous versions. [#121973](https://github.com/cockroachdb/cockroach/pull/121973)
- [Schema changes](https://www.cockroachlabs.com/docs/v24.2/online-schema-changes) that cause a data backfill, such as adding a non-nullable column or changing the primary key, will now split and scatter the temporary indexes used to perform the change. This reduces the chance of causing a write hotspot that can slow down foreground traffic. [#120736](https://github.com/cockroachdb/cockroach/pull/120736)
- Multiple or large numbers of [grants](https://www.cockroachlabs.com/docs/v24.2/show-grants) on tables and types within one [transaction](https://www.cockroachlabs.com/docs/v24.2/transactions) now run faster. [#123414](https://github.com/cockroachdb/cockroach/pull/123414)
- [Lock operations](https://www.cockroachlabs.com/docs/v24.2/architecture/transaction-layer#writes-and-reads-phase-1) are now removed from [query plans](https://www.cockroachlabs.com/docs/v24.2/cost-based-optimizer) when the optimizer can prove that no rows would be locked. [#124830](https://github.com/cockroachdb/cockroach/pull/124830)
- Some privilege checks when scanning the [`crdb_internal.system_jobs`](https://www.cockroachlabs.com/docs/v24.2/crdb-internal) internal table now happen once before the scan, instead of once for each row. This will improve performance for queries that read from `crdb_internal.system_jobs`. [#117438](https://github.com/cockroachdb/cockroach/pull/117438)
- Improved the initial range descriptor scan on startup. [Node startup](https://www.cockroachlabs.com/docs/v24.2/cockroach-start) should now be faster on larger stores. [#123959](https://github.com/cockroachdb/cockroach/pull/123959)
- Previously, concurrent transactions that constructed large write batches could cause [hotspots](https://www.cockroachlabs.com/docs/v24.2/ui-hot-ranges-page). This was because the [transaction record](https://www.cockroachlabs.com/docs/v24.2/architecture/transaction-layer#transaction-records) for all [ranges](https://www.cockroachlabs.com/docs/v24.2/architecture/reads-and-writes-overview#range) would coalesce on a single range, which would then cause this range's [leaseholder](https://www.cockroachlabs.com/docs/v24.2/architecture/reads-and-writes-overview#leaseholder) to perform all intent resolution work. This is fixed by distributing transaction records randomly across the ranges the write batch touches. In turn, hotspots are prevented. [#125744](https://github.com/cockroachdb/cockroach/pull/125744)
