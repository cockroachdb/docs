The `SHOW SAVEPOINT STATUS` [statement](/docs/v25.4/sql-statements) lists the active [savepoints](/docs/v25.4/savepoint) in the current [transaction](/docs/v25.4/transactions).

## Required privileges

No [privileges](/docs/v25.4/security-reference/authorization#managing-privileges) are required to create or show a savepoint. However, privileges are required for each statement within a transaction.

## Response

The following fields are returned for each savepoint.

<table><thead><tr><th>Field</th><th>Description</th></tr></thead><tbody><tr><td><code>savepoint&#95;name</code></td><td>The name of the savepoint.</td></tr><tr><td><code>is&#95;initial&#95;savepoint</code></td><td>Whether the savepoint is the outermost savepoint in the transaction.</td></tr></tbody></table>

## Example

1. Open a [transaction](/docs/v25.4/transactions) using [`BEGIN`](/docs/v25.4/begin-transaction), and create a [nested transaction](/docs/v25.4/transactions#nested-transactions) using a [savepoint](/docs/v25.4/savepoint):

    ```sql
    > BEGIN;
    SAVEPOINT foo;
    ```
2. Use the `SHOW SAVEPOINT STATUS` statement to list the active savepoints in the current nested transaction.

    ```sql
    > SHOW SAVEPOINT STATUS;
    ```

    ```text
      savepoint_name | is_initial_savepoint
    -----------------+-----------------------
      foo            |        true
    (1 row)
    ```

    Currently, there is only one savepoint.
3. Commit this nested transaction by issuing the [`RELEASE SAVEPOINT`](/docs/v25.4/release-savepoint) statement, then clear the connection for the next transaction by issuing a [`COMMIT`](/docs/v25.4/commit-transaction) statement:

    ```sql
    > RELEASE SAVEPOINT foo;
    COMMIT;
    ```

    If we did not want to commit this nested transaction, but restart it instead, we would have issued a [`ROLLBACK TO SAVEPOINT`](/docs/v25.4/rollback-transaction#rollback-a-nested-transaction).
