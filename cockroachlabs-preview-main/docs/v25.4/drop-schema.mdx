The `DROP SCHEMA` [statement](/docs/v25.4/sql-statements) removes a user-defined [schema](/docs/v25.4/sql-name-resolution#naming-hierarchy).

<Note>
The `DROP SCHEMA` statement performs a schema change. For more information about how online schema changes work in CockroachDB, see [Online Schema Changes](/docs/v25.4/online-schema-changes).
</Note>

## Required privileges

The user must have the `DROP` [privilege](/docs/v25.4/security-reference/authorization#managing-privileges) on the schema and on all tables in the schema. If the user is the owner of the schema, `DROP` privileges are not necessary.

## Examples

#### Setup

To follow along, run [`cockroach demo`](/docs/v25.4/cockroach-demo) to start a temporary, in-memory cluster with the [`movr`](/docs/v25.4/movr) sample dataset preloaded:

```shell
$ cockroach demo
```

### Drop a schema

```sql
> CREATE SCHEMA org_one;
```

```sql
> SHOW SCHEMAS;
```

```text
     schema_name
----------------------
  crdb_internal
  information_schema
  org_one
  pg_catalog
  pg_extension
  public
(6 rows)
```

```sql
> DROP SCHEMA org_one;
```

```sql
> SHOW SCHEMAS;
```

```text
     schema_name
----------------------
  crdb_internal
  information_schema
  pg_catalog
  pg_extension
  public
(5 rows)
```

### Drop a schema with tables

To drop a schema that contains tables, you need to use the `CASCADE` keyword.

```sql
> CREATE SCHEMA org_two;
```

```sql
> SHOW SCHEMAS;
```

```text
     schema_name
----------------------
  crdb_internal
  information_schema
  org_two
  pg_catalog
  pg_extension
  public
(6 rows)
```

```sql
> CREATE TABLE org_two.users (
        id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
        city STRING,
        name STRING,
        address STRING
);
```

```sql
> SHOW TABLES FROM org_two;
```

```text
  schema_name | table_name | type  | estimated_row_count
--------------+------------+-------+----------------------
  org_two     | users      | table |                   0
(1 row)
```

```sql
> DROP SCHEMA org_two;
```

```text
ERROR: schema "org_two" is not empty and CASCADE was not specified
SQLSTATE: 2BP01
```

```sql
> DROP SCHEMA org_two CASCADE;
```

```sql
> SHOW SCHEMAS;
```

```text
     schema_name
----------------------
  crdb_internal
  information_schema
  pg_catalog
  pg_extension
  public
(5 rows)
```
