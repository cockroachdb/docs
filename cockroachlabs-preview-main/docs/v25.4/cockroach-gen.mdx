The `cockroach gen` [command](/docs/v25.4/cockroach-commands) can generate command-line interface (CLI) utilities ([`man` pages](https://wikipedia.org/wiki/Man_page) and a `bash` autocompletion script), example SQL data suitable to populate test databases, and an HAProxy configuration file for load balancing a running cluster.

## Subcommands

<table><thead><tr><th>Subcommand</th><th>Usage</th></tr></thead><tbody><tr><td><code>man</code></td><td>Generate man pages for CockroachDB.</td></tr><tr><td><code>autocomplete</code></td><td>Generate <code>bash</code> or <code>zsh</code> autocompletion script for CockroachDB.<br/><br/><strong>Default:</strong><code>bash</code></td></tr><tr><td><code>example-data</code></td><td>Generate example SQL datasets. You can also use the <a href="/docs/v25.4/cockroach-workload"><code>cockroach workload</code></a> command to generate these sample datasets in a persistent cluster and the <a href="/docs/v25.4/cockroach-demo"><code>cockroach demo &lt;dataset&gt;</code></a> command to generate these datasets in a temporary, in-memory cluster.</td></tr><tr><td><code>haproxy</code></td><td>Generate an HAProxy config file for a running CockroachDB cluster. The node addresses included in the config are those advertised by the nodes. Make sure hostnames are resolvable and IP addresses are routable from HAProxy.<br/><br/><a href="/docs/v25.4/node-shutdown?filters=decommission">Decommissioned nodes</a> are excluded from the config file.</td></tr></tbody></table>

## Synopsis

Generate man pages:

```shell
$ cockroach gen man
```

Generate bash autocompletion script:

```shell
$ cockroach gen autocomplete
```

Generate example SQL data:

```shell
$ cockroach gen example-data intro | cockroach sql
```

```shell
$ cockroach gen example-data startrek | cockroach sql
```

Generate an HAProxy config file for a running cluster:

```shell
$ cockroach gen haproxy
```

View help:

```shell
$ cockroach gen --help
```

```shell
$ cockroach gen man --help
```

```shell
$ cockroach gen autocomplete --help
```

```shell
$ cockroach gen example-data --help
```

```shell
$ cockroach gen haproxy --help
```

### Logging

By default, this command logs messages to `stderr`. This includes events with `WARNING` [severity](/docs/v25.4/logging#logging-levels-severities) and higher.

If you need to troubleshoot this command's behavior, you can [customize its logging behavior](/docs/v25.4/configure-logs).

## Examples

### Generate `man` pages

Generate man pages:

```shell
$ cockroach gen man
```

Move the man pages to the man directory:

```shell
$ sudo mv man/man1/* /usr/share/man/man1
```

Access man pages:

```shell
$ man cockroach
```

### Generate a `bash` autocompletion script

Generate bash autocompletion script:

```shell
$ cockroach gen autocomplete
```

Add the script to your `.bashrc` and `.bash_profle`:

```shell
$ printf "\n\n#cockroach bash autocomplete\nsource '<path to>cockroach.bash'" >> ~/.bashrc
```

```shell
$ printf "\n\n#cockroach bash autocomplete\nsource '<path to>cockroach.bash'" >> ~/.bash_profile
```

You can now use `tab` to autocomplete `cockroach` commands.

### Generate example data

### Generate an HAProxy config file

[HAProxy](http://www.haproxy.org/) is one of the most popular open-source TCP load balancers, and CockroachDB includes a built-in command for generating a configuration file that is preset to work with your running cluster.

<Tabs>
<Tab title="Secure">

To generate an HAProxy config file for an entire secure cluster, run the `cockroach gen haproxy` command, specifying the location of [certificate directory](/docs/v25.4/cockroach-cert) and the address of any instance running a CockroachDB node:

```shell
$ cockroach gen haproxy \
--certs-dir=<path to certs directory> \
--host=<address of any node in the cluster>
```

To limit the HAProxy config file to nodes matching specific ["localities"](/docs/v25.4/cockroach-start#locality), use the `--localities` flag, specifying the explicit locality tier(s) or a regular expression to match against:

```shell
$ cockroach gen haproxy \
--certs-dir=<path to certs directory> \
--host=<address of any node in the cluster>
--locality=region=us.*
```

</Tab>
<Tab title="Insecure">

To generate an HAProxy config file for an entire insecure cluster, run the `cockroach gen haproxy` command, specifying the address of any instance running a CockroachDB node:

```shell
$ cockroach gen haproxy \
--insecure \
--host=<address of any node in the cluster>
```

To limit the HAProxy config file to nodes matching specific ["localities"](/docs/v25.4/cockroach-start#locality), use the `--localities` flag, specifying the explicit locality tier(s) or a regular expression to match against:

```shell
$ cockroach gen haproxy \
--insecure \
--host=<address of any node in the cluster>
--locality=region=us.*
```

</Tab>
</Tabs>

By default, the generated configuration file is called `haproxy.cfg` and looks as follows, with the `server` addresses pre-populated correctly:

```text
global
  maxconn 4096

defaults
    mode                tcp
    # Timeout values should be configured for your specific use.
    # See: https://cbonte.github.io/haproxy-dconv/1.8/configuration.html#4-timeout%20connect
    timeout connect     10s
    timeout client      1m
    timeout server      1m
    # TCP keep-alive on client side. Server already enables them.
    option              clitcpka

listen psql
    bind :26257
    mode tcp
    balance roundrobin
    option httpchk GET /health?ready=1
    server cockroach1 <node1 address>:26257 check port 8080
    server cockroach2 <node2 address>:26257 check port 8080
    server cockroach3 <node3 address>:26257 check port 8080
```

The file is preset with the minimal [configurations](http://cbonte.github.io/haproxy-dconv/1.7/configuration) needed to work with your running cluster:

<table><thead><tr><th>Field</th><th>Description</th></tr></thead><tbody><tr><td><code>timeout connect</code><br/><code>timeout client</code><br/><code>timeout server</code></td><td>Timeout values that should be suitable for most deployments.</td></tr><tr><td><code>bind</code></td><td>The port that HAProxy listens on. This is the port clients will connect to and thus needs to be allowed by your network configuration.<br/><br/>This tutorial assumes HAProxy is running on a separate machine from CockroachDB nodes. If you run HAProxy on the same machine as a node (not recommended), you'll need to change this port, as <code>26257</code> is likely already being used by the CockroachDB node.</td></tr><tr><td><code>balance</code></td><td>The balancing algorithm. This is set to <code>roundrobin</code> to ensure that connections get rotated amongst nodes (connection 1 on node 1, connection 2 on node 2, etc.). Check the <a href="http://cbonte.github.io/haproxy-dconv/1.7/configuration#4-balance">HAProxy Configuration Manual</a> for details about this and other balancing algorithms.</td></tr><tr><td><code>option httpchk</code></td><td>The HTTP endpoint that HAProxy uses to check node health. <a href="/docs/v25.4/monitoring-and-alerting#health-ready-1"><code>/health?ready=1</code></a> ensures that HAProxy doesn't direct traffic to nodes that are live but not ready to receive requests.</td></tr><tr><td><code>server</code></td><td>For each included node, this field specifies the address the node advertises to other nodes in the cluster, i.e., the addressed pass in the <a href="/docs/v25.4/cockroach-start#networking"><code>--advertise-addr</code> flag</a> on node startup. Make sure hostnames are resolvable and IP addresses are routable from HAProxy.</td></tr></tbody></table>

<Note>
For full details on these and other configuration settings, see the [HAProxy Configuration Manual](http://cbonte.github.io/haproxy-dconv/1.7/configuration).
</Note>
