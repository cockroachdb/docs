The `RESUME JOB` [statement](/docs/v25.4/sql-statements) lets you resume the following types of jobs:

- [`IMPORT`](/docs/v25.4/import-into) jobs
- [`BACKUP`](/docs/v25.4/backup) and [`RESTORE`](/docs/v25.4/restore) jobs
- [User-created table statistics](/docs/v25.4/create-statistics) jobs
- [Automatic table statistics](/docs/v25.4/cost-based-optimizer#table-statistics) jobs
- [Changefeeds](/docs/v25.4/create-changefeed)
- [Schema change](/docs/v25.4/online-schema-changes) jobs
- [Scheduled backup](/docs/v25.4/manage-a-backup-schedule) jobs

## Required privileges

To resume a job, the user must be a member of the `admin` role or must have the [`CONTROLJOB`](/docs/v25.4/create-user#create-a-user-that-can-pause-resume-and-cancel-non-admin-jobs) [role option](/docs/v25.4/security-reference/authorization#role-options) set. Non-admin users cannot resume admin users' jobs.

For changefeeds, users with the [`CHANGEFEED`](/docs/v25.4/create-changefeed#required-privileges) privilege on a set of tables can resume changefeed jobs running on those tables.

## Examples

### Pause a job

```sql
> SHOW JOBS;
```

```text
      job_id     |  job_type |               description                      |...
-----------------+-----------+------------------------------------------------+...
  27536791415282 |  RESTORE  | RESTORE db.* FROM 'azure-blob://backup/db/tbl' |...
```

```sql
> PAUSE JOB 27536791415282;
```

### Resume a single job

```sql
> RESUME JOB 27536791415282;
```

### Resume multiple jobs

To resume multiple jobs, nest a [`SELECT` clause](/docs/v25.4/select-clause) that retrieves `job_id`(s) inside the `RESUME JOBS` statement:

```sql
> RESUME JOBS (WITH x AS (SHOW JOBS) SELECT job_id FROM x
      WHERE user_name = 'maxroach');
```

All jobs created by `maxroach` will be resumed.

### Resume by job type

To resume all jobs by the type of job, use the `RESUME ALL {job} JOBS` statement. You can resume all `BACKUP`, `RESTORE`, `CHANGEFEED`, `IMPORT` jobs using this statement, for example:

```sql
RESUME ALL BACKUP JOBS;
```

### Resume jobs for a schedule

To resume jobs for a specific [backup schedule](/docs/v25.4/create-schedule-for-backup), use the schedule's `id`:

```sql
> RESUME JOBS FOR SCHEDULE 590204387299262465;
```

```text
RESUME JOBS FOR SCHEDULES 1
```

You can also resume multiple schedules by nesting a [`SELECT` clause](/docs/v25.4/select-clause) that retrieves `id`(s) inside the `PAUSE JOBS` statement:

```sql
> RESUME JOBS FOR SCHEDULES WITH x AS (SHOW SCHEDULES) SELECT id FROM x WHERE label = 'test_schedule';
```

```text
RESUME JOBS FOR SCHEDULES 2
```
