User-defined functions and stored procedures support SECURITY DEFINER

You can create or alter a [user-defined function (UDF)]({% link v24.3/user-defined-functions.md %}) or [stored procedure]({% link v24.3/stored-procedures.md %}) with [EXTERNAL] SECURITY DEFINER instead of the default [EXTERNAL] SECURITY INVOKER. With [`[SECURITY DEFINER]`]({% link v24.3/create-function.md %}#create-a-security-definer-function), the privileges of the owner are checked when the UDF or stored procedure is executed, rather than the privileges of the executor. The EXTERNAL keyword is optional and exists for SQL language conformity.

CockroachDB now supports triggers in Preview

CockroachDB now supports [triggers]({% link v24.3/triggers.md %}). Triggers allow automatic execution of specified functions in response to specified events on a particular table or view. They can be used for automating tasks, enforcing business rules, and maintaining data integrity. This feature is in Preview.
