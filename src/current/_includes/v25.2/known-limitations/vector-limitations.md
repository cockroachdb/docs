- Creating a vector index through a backfill disables mutations ([`INSERT`]({% link {{ page.version.version }}/insert.md %}), [`UPSERT`]({% link {{ page.version.version }}/upsert.md %}), [`UPDATE`]({% link {{ page.version.version }}/update.md %}), [`DELETE`]({% link {{ page.version.version }}/delete.md %})) on the table.
- `IMPORT INTO` is not supported on tables with vector indexes. You can import the vectors first and create the index after import is complete.
- Only L2 distance (`<->`) searches are accelerated.
- Index acceleration with filters is only supported if the filters match prefix columns.
- Index recommendations are not provided for vector indexes.
- Vector index queries may return incorrect results when the underlying table uses multiple column families.
