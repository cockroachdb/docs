### General Changes

- disabling diagnostics reporting also disables new version notification checks. [#23008][#23008]

### Admin UI Changes

- Fixes an issue where decommissioned nodes are
  included in cluster stats aggregates. [#22864][#22864]

### Bug Fixes

- Avoid disruptions in performance when gracefully
  shutting a node down. [#23302][#23302]
- Fix a zero qps scenario when a node would be
  gracefully drained. [#23306][#23306]
- do not crash if COCKROACH_UPDATE_CHECK_URL or COCKROACH_USAGE_REPORT_URL overrides are empty. [#23008][#23008]
- the conversion from `INTERVAL` to `FLOAT` now
  properly returns the number of seconds in the interval. [#22893][#22893]
- Queries over virtual tables with an explicit
  database name prefix (e.g. `select * from mydb.crdb_internal.tables`)
  would not be scrubbed properly from reported statistics. This is now
  fixed. [#22815][#22815]
- A `COMMIT` reporting an error generated by a
  previous parallel (i.e. `RETURNING NOTHING`) statement no longer leaves
  the connection in an aborted transaction state. Instead the transaction
  is considered completed and a `ROLLBACK` is not necessary. [#22814][#22814]
- Fixed issue with stale buffer data when using the binary
  format for arrays. [#22262][#22262]
